  <files>app/src/lib/wholesale-score.ts, app/src/lib/wholesale-parser.ts, app/src/lib/wholesale-actions.ts, app/src/lib/wholesale-queries.ts</files>

Create 4 new lib files:

**wholesale-score.ts** -- Pure function, no DB, no imports beyond types:
- `computeWholesaleScore(arv, repairEstimate, askingPrice, wholesaleFee = 15000): WholesaleScoreBreakdown`
- Factor 1 (40% weight): MAO spread ratio. spreadRatio = spreadDollars / MAO. >=0.15 -> 10pts, >=0.05 -> 7pts, >=0 -> 5pts, >=-0.05 -> 2pts, else 0.
- Factor 2 (30% weight): Equity % of ARV. spreadPct = spreadDollars / ARV. >=0.15 -> 10pts, >=0.10 -> 7pts, >=0.05 -> 4pts, else 0.
- Factor 3 (30% weight): End buyer ROI. endBuyerAllIn = askingPrice + repairEstimate. endBuyerProfit = ARV - endBuyerAllIn. ROI% = endBuyerProfit / endBuyerAllIn * 100. >=20% -> 10pts, >=15% -> 7pts, >=10% -> 4pts, else 0.
- total = round(factor1*0.4 + factor2*0.3 + factor3*0.3), clamped 1-10
- verdict: >=7 green, >=4 yellow, else red
- Return full breakdown object

**wholesale-parser.ts** -- Regex parser for structured wholesaler email blasts:
- `parseWholesaleEmail(text, fromEmail, subject): ParsedWholesaleDeal`
- Parse address (first line before "ASKING" or newline)
- Parse asking price ("ASKING $XXK" or "ASKING $XXX,XXX") using parseDollars helper that handles K-notation and commas
- Parse ARV ("ARV: $XXK" or "ARV: $XXX,XXX")
- Parse sqft ("Sq Ft: 1,328" etc)
- Parse beds/baths ("Beds: 2" / "Baths: 1.5")
- Parse year built ("Year Built: 1972")
- Parse tax ID ("Tax ID: XX-XXXX-XXXX")
- Parse contact info ("Contact Name @ (801) 555-1234")
- `confidence` = count of non-null fields / total_fields (0-1 ratio)
- `normalizeAddress(addr)`: lowercase, remove apt/unit/#, strip punctuation, trim
- All fields return null on parse failure (never throw)

**wholesale-actions.ts** -- "use server" server actions:
- `createWholesaleLead(formData: FormData)`: validates with zod, computes score via computeWholesaleScore, normalizes address, upserts wholesaler by email if provided, inserts wholesale_leads row. Returns `{ id }`.
- `updateWholesaleLead(formData: FormData)`: updates lead fields, recomputes score, returns void.
- `updateWholesaleLeadStatus(wholesaleLeadId: string, newStatus: string, note?: string)`: updates status, auto-logs a status_change note. Returns void.
- `addWholesaleNote(formData: FormData)`: inserts into wholesaleLeadNotes. Returns void.
- `upsertWholesaler(email: string | null, name: string | null): Promise<string | null>`: onConflictDoUpdate on email, returns wholesaler id. Internal helper.
- `createWholesaleLeadFromEmail(bodyText: string, fromEmail: string, subject: string)`: calls parseWholesaleEmail, calls upsertWholesaler, inserts lead with status "new", parsedDraft = JSON of parsed fields, rawEmailText = bodyText. Returns `{ id }`.

**wholesale-queries.ts** -- DB query functions:
- `getWholesaleLeads(filters?: { status?, verdict?, wholesalerId? })`: returns WholesaleLeadWithWholesaler[] with left join to wholesalers. Ordered by createdAt desc.
- `getWholesaleLead(id: string)`: single lead with wholesaler join.
- `getWholesaleLeadNotes(wholesaleLeadId: string)`: notes ordered by createdAt desc.
- `getWholesalers()`: all wholesalers ordered by name.
- `getWholesalerStats(wholesalerId: string)`: totalSent (count leads), totalPromoted (count where status=promoted), avgSpread (avg of mao - askingPrice where both non-null).
- `getWholesalersWithStats()`: all wholesalers with their aggregate stats (two-query + post-merge pattern from project convention).
- `checkDuplicateAddress(addressNormalized: string, excludeId?: string)`: returns existing leads with same normalized address for duplicate warning.

All server actions follow existing patterns: auth check via `await auth()`, zod v4 validation, revalidatePath("/wholesale").
