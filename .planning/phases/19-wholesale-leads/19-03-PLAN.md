**api/inbound/route.ts** -- API route handler (NOT a server action):
- `export async function POST(req: Request)` -- handles Resend `email.received` webhook
- Parse JSON payload. If `payload.type !== "email.received"`, return `Response.json({ ok: true })` immediately.
- Extract `emailId` from `payload.data.email_id`.
- Fetch full email body from Resend API: `GET https://api.resend.com/emails/{emailId}` with Authorization header `Bearer ${process.env.RESEND_API_KEY}`. Use native fetch (no resend SDK needed for this endpoint).
- Extract `text` field from response (plain text body). Fallback to `html` if text is empty (strip HTML tags with basic regex).
- Extract `from` email address from payload.data.from or response.
- Extract `subject` from payload.data.subject or response.
- Call `createWholesaleLeadFromEmail(bodyText, fromEmail, subject)` from wholesale-actions.ts.
- Return `Response.json({ ok: true })` with status 200. Must respond quickly (webhook timeout).
- Wrap entire handler in try/catch -- on error, log and still return 200 (prevent Resend retries on processing errors).
- NO auth check -- this is a public webhook endpoint. Resend doesn't send a verification signature, so accept all POST requests. The webhook URL itself is the secret.

**wholesale-parse-review.tsx** -- "use client" component:
- Props: `{ lead: WholesaleLeadWithWholesaler }` (for leads with status "new" that have parsedDraft)
- Shows a pre-filled form with all fields from the parsedDraft JSON (parse it)
- Raw email text shown in a collapsible section so user can reference the original
- Parse confidence indicator: "X/12 fields extracted" with colored badge
- Duplicate warning: if lead has same addressNormalized as another lead, show banner "Similar address found: [address] from [wholesaler] on [date]"
- "Save & Analyze" button calls `updateWholesaleLead` server action with corrected fields, changes status to "analyzing"
- Live WholesaleAnalysis preview below the form (reuses wholesale-analysis.tsx from Plan 02)
- Form layout: two-column on desktop, single column on mobile
