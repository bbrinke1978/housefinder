Create a new file with these exports:

1. **`listFeedbackItems(filters)`** — accepts `{ status?, type?, priority?, assigneeId?, reporterId?, search?, includeDeleted? }`. Returns array of items joined with reporter name + assignee name (LEFT JOIN users). FTS via Postgres `to_tsquery('english', $1)` if `search` is provided. Sort: open-status-first (status NOT IN shipped/wontfix/duplicate), then priority desc, then created_at desc. Excludes `deleted_at IS NOT NULL` unless `includeDeleted` true. Cap at 200 rows.

2. **`getFeedbackItemDetail(id)`** — returns the item plus arrays of joined comments (with author name), attachments (with uploader name + SAS URL generated server-side), and activity entries (with actor name). Uses Drizzle's parallel `Promise.all` pattern for the four selects. Excludes deleted child rows from the comment/attachment arrays. Returns `null` if item missing or its `deleted_at` is set.

3. **`countOpenFeedbackForUser(userId)`** — small helper for the nav badge: returns count of items where status NOT IN ('shipped','wontfix','duplicate') AND assigneeId = userId AND deletedAt IS NULL. Used by Plan 03 for the nav-bar badge.

All functions use the existing `db` from `@/db/client` and Drizzle query builder. NO authentication checks here (queries are pure reads; auth happens at the action layer in Task 3). Imports: `feedbackItems, feedbackComments, feedbackAttachments, feedbackActivity, users` from `@/db/schema`.

Generate SAS URLs only on detail-page reads (one-hour TTL is plenty for a viewing session). Don't generate for list view (avoids hot-loop overhead).

Auth: any signed-in user can edit fields BUT only the reporter or admin can edit `title`/`description` more than 5 minutes after creation (lock-down for accountability). 3. **`updateFeedbackStatus(id, newStatus, shipNote?)`** — auth: only Brian (admin gate via `isAdmin(session)` helper) can set status to 'shipped', 'wontfix', or 'duplicate'. Inserts `feedback_activity` row. If status='shipped' and a shipNote is provided, store it in the activity row's `new_value` JSON (`{ status: 'shipped', note: '...' }`).

4. **`createFeedbackComment(itemId, body)`** — auth: any signed-in user. INSERT feedback_comments. INSERT feedback_activity with action='comment_added' and new_value=commentId. Returns `{ id }`.

<task type="auto">
  <name>Task 4: Create attachment upload API route</name>
  <files>app/src/app/api/feedback/[id]/attachments/route.ts</files>
  <action>
Create a Next.js App Router POST handler:

```typescript
import { NextResponse } from "next/server";
import { auth } from "@/auth";
import { db } from "@/db/client";
import { feedbackItems, feedbackAttachments, feedbackActivity } from "@/db/schema";
import { eq } from "drizzle-orm";
import { uploadFeedbackBlob, generateFeedbackSasUrl } from "@/lib/blob-storage";
import { randomUUID } from "node:crypto";

export const runtime = "nodejs";

export async function POST(req: Request, { params }: { params: Promise<{ id: string }> }) {
  const session = await auth();
  if (!session?.user?.id) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  const { id: itemId } = await params;
  // Verify the parent item exists and isn't soft-deleted
  const [item] = await db.select().from(feedbackItems).where(eq(feedbackItems.id, itemId)).limit(1);
  if (!item || item.deletedAt) return NextResponse.json({ error: "Not found" }, { status: 404 });

  const formData = await req.formData();
  const file = formData.get("file") as File | null;
  const commentId = formData.get("commentId") as string | null;
  if (!file) return NextResponse.json({ error: "Missing file" }, { status: 400 });

  // Validate
  if (!file.type.startsWith("image/")) return NextResponse.json({ error: "Only images allowed" }, { status: 400 });
  if (file.size > 5 * 1024 * 1024) return NextResponse.json({ error: "Max 5MB" }, { status: 400 });

  // Upload
  const buffer = Buffer.from(await file.arrayBuffer());
  const attachmentId = randomUUID();
  const ext = file.name.split(".").pop() || "png";
  const blobName = `${itemId}/${attachmentId}.${ext}`;
  await uploadFeedbackBlob(buffer, blobName, file.type);

  // Insert + audit
  await db.insert(feedbackAttachments).values({
    id: attachmentId,
    itemId,
    commentId: commentId || null,
    blobName,
    mimeType: file.type,
    sizeBytes: file.size,
    uploadedBy: session.user.id,
  });
  await db.insert(feedbackActivity).values({
    itemId,
    actorId: session.user.id,
    action: "attachment_added",
    newValue: attachmentId,
  });

  return NextResponse.json({ id: attachmentId, sasUrl: generateFeedbackSasUrl(blobName) });
}
```

Also add a corresponding `DELETE` handler at `app/src/app/api/feedback/[id]/attachments/[attachmentId]/route.ts` that soft-deletes the attachment (uploader OR admin).
