<task type="auto">
  <name>Task 5: Status controls component</name>
  <files>app/src/components/feedback/feedback-status-controls.tsx</files>
  <action>
Props: `{ item: FeedbackItem; users: { id, name, email }[]; isAdmin: boolean; currentUserId: string }`.

Three dropdowns (shadcn Select):
- **Status**: options depend on `isAdmin`. Non-admins see `new, planned, in_progress`. Admins also see `shipped, wontfix, duplicate`.
- **Priority**: any signed-in user can change.
- **Assignee**: select from users list. "Unassigned" option = null.

On change, call the corresponding server action: `updateFeedbackItem({ priority, assigneeId })` or `updateFeedbackStatus(id, newStatus, shipNote?)`. Show a confirm dialog if changing to `shipped` or `wontfix` ("Are you sure?") with an optional shipNote textarea for shipped.

Optimistic UI: show the new value immediately, revert on server error.
  </action>
