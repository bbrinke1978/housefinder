1. analytics-activity-log.tsx (ANALYTICS-07):
   - Props: `{ data: ActivityEntry[] }`
   - Empty state: "No activity recorded yet. Notes, status changes, and calls will appear here as you work."
   - Render as a vertical timeline/list — each entry is a row showing:
     - Left: colored icon (blue circle for notes, green phone icon for calls, yellow arrow for status changes — based on type + sub_type).
     - Center: description text. For "note" type show the note text (truncated to 100 chars). For "call" type show "Called — {outcome}" formatted. For status_change sub_type show the note text (which already says "Status changed from X to Y").
     - Right side: timestamp formatted as "Mar 26, 2:30 PM" using date-fns `format(new Date(created_at), "MMM d, h:mm a")`.
     - Below description: property address + city in muted text, linked to `/properties/{lead_id}` (or just displayed if no link desired — keep it simple).
   - Show most recent 50 entries (already limited by query).
   - Style: `divide-y` list with `py-3` per entry. Mobile-friendly (stacked on small screens).

2. app/api/export/route.ts (ANALYTICS-08):
   - `export const dynamic = "force-dynamic";`
   - GET handler accepting `?type=leads|deals|buyers` searchParam.
   - Auth check: `const session = await auth(); if (!session?.user) return new NextResponse("Unauthorized", { status: 401 });`
   - Import auth from `@/auth` (same pattern as other API routes in the project).
   - Switch on type:
     - "leads": call getPropertiesForExport(), filename `leads-{YYYY-MM-DD}.csv`
     - "deals": call getDealsForExport(), filename `deals-{YYYY-MM-DD}.csv`
     - "buyers": call getBuyersForExport(), filename `buyers-{YYYY-MM-DD}.csv`
     - default: return 400 "Invalid export type"
   - buildCsv helper function: take rows array, extract headers from Object.keys(rows[0]), build CSV string with headers row + data rows. Use JSON.stringify per cell to handle commas/quotes/newlines. If rows empty, return just headers or empty string.
   - Return `new NextResponse(csv, { headers: { "Content-Type": "text/csv", "Content-Disposition": `attachment; filename="${filename}"` } })`.
