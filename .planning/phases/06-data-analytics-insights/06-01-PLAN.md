  <action>
1. In app/ directory: `npm install recharts react-is@19.1.0`. Add `"overrides": { "react-is": "19.1.0" }` to app/package.json top level (required for React 19 compatibility — without this, recharts renders blank charts).

2. Add to app/src/db/schema.ts (after dealNotes table):
   - `callOutcomeEnum` pgEnum with values: "answered", "voicemail", "no_answer", "wrong_number"
   - `callLogs` pgTable "call_logs" with columns: id (uuid pk), leadId (uuid FK to leads.id, notNull), outcome (callOutcomeEnum, notNull), source (text, nullable — "manual" | "tracerfy" etc), durationSeconds (integer, nullable), notes (text, nullable), calledAt (timestamp tz, notNull, defaultNow), createdAt (timestamp tz, notNull, defaultNow)
   - Indexes: idx_call_logs_lead_id on leadId, idx_call_logs_called_at on calledAt
   - Export `CallLogRow = InferSelectModel<typeof callLogs>`

3. Generate migration: `npx drizzle-kit generate` from app/ directory.

4. Create app/src/lib/analytics-queries.ts with ALL analytics query functions. Use `db.execute<T>(sql`...`)` pattern from existing queries.ts. Every function returns typed data:

   - `getPipelineFunnelData()`: SELECT status, COUNT(*)::int, AVG(EXTRACT(EPOCH FROM (updated_at - created_at)) / 86400)::numeric(10,1) FROM leads GROUP BY status ORDER BY CASE status WHEN 'new' THEN 1 WHEN 'contacted' THEN 2 WHEN 'follow_up' THEN 3 WHEN 'closed' THEN 4 WHEN 'dead' THEN 5 ELSE 6 END. Returns FunnelStage[].

   - `getMarketComparisonData()`: GROUP BY p.city from leads JOIN properties — total leads, hot leads (is_hot=true), converted (status='closed'). Compute conversionRate. Returns MarketStat[].

   - `getPropertyTrendData()`: date_trunc('week', l.first_seen_at)::date::text, p.city, COUNT(*)::int from leads JOIN properties WHERE first_seen_at > NOW() - INTERVAL '6 months'. Returns TrendPoint[].

   - `getScraperHealthData()`: SELECT * from scraperHealth ordered by county. Compute status (green/yellow/red) and freshnessHours in TypeScript per Pattern 7 in research. Returns ScraperHealthRow[].

   - `getLeadSourceAttribution()`: JOIN distress_signals -> leads -> LEFT JOIN deals via propertyId. GROUP BY signal_type. Returns AttributionStat[].

   - `getOutreachStats()`: COUNT from call_logs GROUP BY outcome. Also compute contactRate (answered / total). Returns OutreachStat[].

   - `getRecentActivityLog(limit=50)`: UNION ALL of lead_notes (type='note') and call_logs (type='call') with address/city from properties JOIN. ORDER BY created_at DESC LIMIT. Returns ActivityEntry[].

   - `getPropertiesForExport()`: SELECT p.*, l.status, l.distress_score, l.is_hot FROM properties p LEFT JOIN leads l ON ... for CSV export. Returns flat rows.

   - `getDealsForExport()`: SELECT d.* FROM deals for CSV export.

   - `getBuyersForExport()`: SELECT * FROM buyers WHERE is_active = true for CSV export.

Export all interfaces (FunnelStage, MarketStat, TrendPoint, ScraperHealthRow, HealthStatus, AttributionStat, OutreachStat, ActivityEntry) and all functions.
  </action>
