  <action>
**Step 1 — Diagnostic: sample 5 parcel IDs from the DB per county** before the full run.

Run this SQL via psql or the app DB client to compare stored format against what UGRC returns:
```sql
SELECT parcel_id, county FROM properties
WHERE county IN ('carbon', 'emery', 'juab', 'millard')
  AND parcel_id IS NOT NULL
ORDER BY county, id
LIMIT 20;
```

Then fetch 3 raw UGRC records for Carbon County to see PARCEL_ID format:
```bash
curl -s "https://services1.arcgis.com/99lidPhWCzftIe9K/arcgis/rest/services/Parcels_Carbon_LIR/FeatureServer/0/query?where=1%3D1&outFields=PARCEL_ID&resultRecordCount=3&f=json" | node -e "const d=JSON.parse(require('fs').readFileSync('/dev/stdin','utf8')); d.features.forEach(f=>console.log(f.attributes.PARCEL_ID));"
```

Compare the two formats. If they differ (e.g., DB has `XX-XXXX-XXXX` but UGRC has `XXXXXXXXXXXX`), the normalization added in Plan 01 handles it. Note the format delta in the summary.

**Step 2 — Run the import script:**
```bash
cd app
DATABASE_URL="$DATABASE_URL" node src/scripts/import-ugrc-assessor.mjs 2>&1 | tee /tmp/ugrc-import-run.log
```

If DATABASE_URL is not in the shell environment, the script will exit with the guard message from Plan 01. Set it before running.

**Step 3 — Capture match rates from output.** Record the per-county and total counts from the script output:
- Updated (enriched) count
- Skipped (no UGRC data to set) count
- No match in our DB count

A low "No match" count per county relative to total unique parcels is expected (our DB only holds distress-signaled properties, not all parcels). A "No match" count near 100% of parcels means normalization did not work — in that case, check the diagnostic output and adjust the normalizeParcelId() strip pattern if the format uses characters other than hyphens/spaces/dots.

**If updated count is zero for all counties:** The normalization is failing. Run:
```sql
SELECT p.parcel_id, p.county FROM properties p LIMIT 5;
```
And compare raw values to UGRC output from step 1. Adjust the normalizeParcelId() regex in the script to match the actual delimiter used, then re-run.
