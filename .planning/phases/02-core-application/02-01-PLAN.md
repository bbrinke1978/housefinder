  <action>
    1. Create the Next.js 15 app at `app/` directory:
       ```bash
       npx create-next-app@15 app --typescript --tailwind --eslint --app --src-dir --no-import-alias
       ```
       If create-next-app prompts interactively, use flags to avoid prompts. The `--no-import-alias` flag prevents the "@/" alias prompt — we will configure it manually.

    2. Install all dependencies (run from `app/` directory):
       ```bash
       npm install next-auth@beta drizzle-orm pg next-themes @hello-pangea/dnd date-fns zod bcryptjs
       npm install -D drizzle-kit @types/pg @types/bcryptjs
       ```
       Then init shadcn:
       ```bash
       npx shadcn@latest init -d
       ```
       Then add required shadcn components:
       ```bash
       npx shadcn@latest add sidebar button input label card badge tabs separator dropdown-menu
       ```

    3. Copy `scraper/src/db/schema.ts` into `app/src/db/schema.ts`. Then extend the schema:
       - Add `lastContactedAt: timestamp("last_contacted_at", { withTimezone: true })` to the `leads` table definition
       - Add a new `leadNotes` table:
         ```typescript
         export const leadNotes = pgTable("lead_notes", {
           id: uuid("id").defaultRandom().primaryKey(),
           leadId: uuid("lead_id").notNull().references(() => leads.id),
           noteText: text("note_text").notNull(),
           noteType: text("note_type").notNull().default("user"), // "user" | "status_change"
           previousStatus: text("previous_status"),
           newStatus: text("new_status"),
           createdAt: timestamp("created_at", { withTimezone: true }).notNull().defaultNow(),
         }, (table) => [
           index("idx_lead_notes_lead_id").on(table.leadId),
         ])
         ```
       - IMPORTANT: The web app schema uses standard TypeScript imports (NOT .js extensions like the scraper's ESM). Next.js handles module resolution.

    4. Create `app/src/db/client.ts` — same pattern as scraper but WITHOUT .js import extensions:
       ```typescript
       import { Pool } from "pg"
       import { drizzle } from "drizzle-orm/node-postgres"
       import * as schema from "./schema"
       const pool = new Pool({
         connectionString: process.env.DATABASE_URL,
         ssl: { rejectUnauthorized: true },
         max: 3,
         idleTimeoutMillis: 300000,
         connectionTimeoutMillis: 5000,
       })
       export const db = drizzle({ client: pool, schema })
       ```

    5. Create `app/drizzle.config.ts` pointing to `src/db/schema.ts` with DATABASE_URL from env.

    6. Generate and run the Drizzle migration to add `lastContactedAt` to leads and create `lead_notes` table:
       ```bash
       npx drizzle-kit generate
       npx drizzle-kit migrate
       ```
       NOTE: This runs against the live Azure PostgreSQL database via DATABASE_URL. The migration is safe — lastContactedAt is nullable (no default needed) and lead_notes is a new table.

    7. Create `app/src/auth.ts` — Auth.js v5 Credentials provider:
       - Use `NextAuth` from "next-auth" with Credentials provider from "next-auth/providers/credentials"
       - JWT session strategy with 30-day maxAge
       - authorize function: validate email against AUTH_EMAIL env var, compare password with bcryptjs against AUTH_PASSWORD_HASH env var
       - signIn page: "/login"
       - Export: `{ handlers, auth, signIn, signOut }`

    8. Create `app/src/middleware.ts`:
       ```typescript
       export { auth as middleware } from "@/auth"
       export const config = {
         matcher: ["/((?!api/auth|login|_next/static|_next/image|favicon.ico).*)"],
       }
       ```

    9. Create `app/src/app/api/auth/[...nextauth]/route.ts`:
       ```typescript
       import { handlers } from "@/auth"
       export const { GET, POST } = handlers
       ```

    10. Create `app/src/app/login/page.tsx` — a server component login page with a client component login form:
        - Simple centered card with email + password inputs
        - Form calls signIn("credentials", { email, password, redirectTo: "/" })
        - Error state for invalid credentials
        - Use shadcn Card, Input, Label, Button components
        - Mobile-friendly with large tap targets (min-h-12 for inputs/buttons)

    11. Create `app/src/types/index.ts` with shared TypeScript types:
        ```typescript
        export type LeadStatus = "new" | "contacted" | "follow_up" | "closed" | "dead"
        export type NewLeadStatus = "new" | "unreviewed"
        export type SignalType = "nod" | "tax_lien" | "lis_pendens" | "probate" | "code_violation" | "vacant"
        export type SignalStatus = "active" | "resolved"

        export interface PropertyWithLead {
          id: string
          parcelId: string
          address: string
          city: string
          state: string
          zip: string | null
          county: string
          ownerName: string | null
          ownerType: string | null
          propertyType: string | null
          distressScore: number
          isHot: boolean
          leadStatus: LeadStatus
          newLeadStatus: NewLeadStatus
          firstSeenAt: Date | null
          lastViewedAt: Date | null
          lastContactedAt: Date | null
        }

        export interface DistressSignalRow {
          id: string
          signalType: SignalType
          status: SignalStatus
          recordedDate: string | null
          sourceUrl: string | null
          createdAt: Date
          resolvedAt: Date | null
        }

        export interface LeadNote {
          id: string
          leadId: string
          noteText: string
          noteType: "user" | "status_change"
          previousStatus: string | null
          newStatus: string | null
          createdAt: Date
        }
        ```

    12. Set `app/next.config.ts` with `output: "standalone"` for Azure deployment.

    13. Create `app/.env.local` with placeholder values:
        ```
        DATABASE_URL=postgresql://... (copy from scraper)
        AUTH_SECRET=<generate with openssl rand -base64 32>
        AUTH_EMAIL=investor@example.com
        AUTH_PASSWORD_HASH=<generate with: node -e "console.log(require('bcryptjs').hashSync('changeme', 10))">
        ```
        Generate actual AUTH_SECRET with: `openssl rand -base64 32`
        Generate actual AUTH_PASSWORD_HASH with: `node -e "console.log(require('bcryptjs').hashSync('temppassword123', 10))"`
