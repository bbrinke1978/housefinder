  <action>
    1. Create `app/src/lib/queries.ts` with these query functions:

       **getDashboardStats()** — returns { total, hot, newToday, needsFollowUp }
       - Use Drizzle sql template with `count(*) filter (where ...)` pattern
       - total: count of all leads
       - hot: count where leads.isHot = true
       - newToday: count where leads.firstSeenAt > now() - interval '24 hours'
       - needsFollowUp: count where leads.status = 'follow_up'

       **getProperties(params)** — accepts filter/sort params, returns PropertyWithLead[]
       - Inner join properties + leads on leads.propertyId = properties.id
       - Filter conditions (all optional, combine with `and()`):
         - city: ilike match on properties.city
         - distressType: exists subquery on distressSignals where signalType matches
         - hot: eq(leads.isHot, true) when hot="true"
         - status: eq(leads.status, value) when provided
       - Sort options via `sort` param:
         - "score" (default): desc(leads.distressScore)
         - "date": desc(leads.firstSeenAt)
         - "city": asc(properties.city)
       - Limit 100 results
       - CRITICAL: In Next.js 15, searchParams is a Promise. The page component must `await searchParams` before passing to this function.

       **getDistinctCities()** — returns string[] of distinct city values from properties table, for filter dropdown.

    2. Create `app/src/components/stats-bar.tsx` — server component (no "use client"):
       - Accepts stats prop: { total, hot, newToday, needsFollowUp }
       - Four stat cards in a row (grid cols-2 on mobile, cols-4 on desktop)
       - Each card shows label + count with appropriate icon and color
       - Hot leads: red/orange accent
       - New today: blue accent
       - Needs follow-up: yellow accent
       - Use shadcn Card component
  </action>
