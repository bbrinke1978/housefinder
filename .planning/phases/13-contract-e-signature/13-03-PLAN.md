<task type="auto">
  <name>Task 1: Signature canvas component</name>
  <files>
    app/src/components/signature-canvas.tsx
  </files>
  <action>
    Create signature-canvas.tsx: "use client" component.
    Props: onSubmit: (signatureData: string, signatureType: "drawn" | "typed") => void, isPending: boolean

    Two modes toggled by tabs/buttons: "Draw" and "Type".

    DRAW MODE:
    - HTML canvas element (400x200 on desktop, full width on mobile with aspect ratio maintained)
    - style={{ touchAction: 'none' }} on canvas to prevent iOS Safari scroll interference (critical pitfall from research)
    - Use Pointer Events API (pointerdown, pointermove, pointerup) for cross-device compatibility (works on mouse, touch, and stylus)
    - Drawing: track isDrawing state. On pointerdown: beginPath + moveTo. On pointermove: lineTo + stroke (2px black line). On pointerup: stop drawing.
    - "Clear" button resets canvas via clearRect
    - On submit: canvas.toDataURL("image/png") produces base64 signature data
    - Gray border, rounded corners, "Sign here" placeholder text centered when empty

    TYPE MODE:
    - Text input field styled to look like a signature (cursive font — use a CSS font-family fallback like "Brush Script MT, cursive")
    - Preview of typed name rendered below input in signature font style
    - On submit: signatureData = typed text, signatureType = "typed"

    Submit button: "Confirm Signature" — disabled when canvas is blank (track hasDrawn state) or typed name is empty. Shows spinner when isPending.

    Responsive: canvas scales to container width on mobile. Use ref to get container width and set canvas dimensions accordingly on mount and resize.
  </action>
