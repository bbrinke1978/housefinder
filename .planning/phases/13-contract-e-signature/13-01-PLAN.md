  <action>
    1. Create contract-queries.ts with these query functions (all follow existing drizzle pattern from deal-queries.ts — direct joins, no relations()):
       - getDealContracts(dealId: string): Promise&lt;ContractWithSigners[]&gt; — fetch all contracts for a deal with their signers, parse clauses JSON, order by createdAt desc
       - getContractById(id: string): Promise&lt;ContractWithSigners | null&gt; — single contract with signers and parsed clauses
       - getContractBySigningToken(token: string): Promise&lt;{ contract: ContractRow; signer: ContractSignerRow } | null&gt; — lookup by signing token, check not expired and not already signed
       - getAllContracts(): Promise&lt;ContractWithSigners[]&gt; — all contracts across all deals for global contracts page, ordered by updatedAt desc
       - getContractCountByDealId(dealId: string): Promise&lt;number&gt; — count for badge display

    2. Create contract-actions.ts with "use server" directive (follow existing deal-actions.ts pattern — zod/v4 validation, revalidatePath):
       - createContract(formData: FormData): Promise&lt;{ id: string } | { error: string }&gt;
         Accepts: dealId, contractType, propertyAddress, city, county, parcelId, sellerName, buyerName, purchasePrice, arv, assignmentFee, earnestMoney, inspectionPeriodDays, closingDays, clauses (JSON string)
         Creates contract row + 2 signer rows (order 1 = seller/buyer depending on type, order 2 = wholesaler/you). Generate signingToken via crypto.randomUUID() for each signer. Set tokenExpiresAt to null initially (set when sent). Signer names/emails come from form data (signerOneName, signerOneEmail, signerTwoName, signerTwoEmail).
         revalidatePath for deal detail page.
       - sendForSigning(contractId: string): Promise&lt;{ success: true } | { error: string }&gt;
         Updates contract status from draft to sent. Sets tokenExpiresAt = now + 72 hours on first signer only. Sets sentAt. Sends signing invitation email to first signer via Resend (use existing mail settings from scraperConfig). revalidatePath.
       - submitSignature(token: string, signatureData: string, signatureType: "drawn" | "typed"): Promise&lt;{ success: true } | { error: string }&gt;
         Verifies token (exists, not expired, not signed). Gets IP from headers(). Generates current PDF bytes via renderToBuffer, hashes with SHA-256. Records signature data, type, IP, userAgent, documentHash. Calls advanceContractStatus helper. Returns success or error.
       - advanceContractStatus(contractId: string): internal helper (not exported as server action)
         Checks all signers. If signer order 1 is signed and order 2 is not: set contract status to seller_signed (or buyer_signed contextually — just use "seller_signed" for purchase_agreement and "countersigned" for assignment where first signer = wholesaler). Set tokenExpiresAt on signer 2 to now + 72 hours. Send signing invitation to signer 2 via Resend.
         If both signers signed: set status to executed, set executedAt. Generate final PDF with audit trail, upload to blob storage via uploadContract(), set signedPdfBlobName. Auto-advance deal status to under_contract (UPDATE deals SET status = 'under_contract' WHERE id = $dealId AND status != 'closed' AND status != 'dead'). Auto-stop campaign enrollments (set status = 'stopped', stopReason = 'contract_executed' WHERE propertyId matches and status = 'active'). Send final signed PDF email to both parties via Resend.
       - voidContract(contractId: string, reason: string): Promise&lt;{ success: true } | { error: string }&gt;
         Sets status to voided, voidedAt, voidReason. revalidatePath.
       - resendSigningLink(signerId: string): Promise&lt;{ success: true } | { error: string }&gt;
         Resets tokenExpiresAt to now + 72 hours. Re-sends signing invitation email to signer.
       - extendSigningDeadline(signerId: string, hours: number): Promise&lt;{ success: true } | { error: string }&gt;
         Sets tokenExpiresAt to now + hours. revalidatePath.

    3. Create contract-pdf.tsx — server-only file with @react-pdf/renderer components:
       - Register Inter font using a local file path: Font.register({ family: "Inter", src: process.cwd() + "/public/fonts/Inter-Regular.ttf" }). Download Inter-Regular.ttf to app/public/fonts/ during task execution (curl from Google Fonts).
       - PurchaseAgreementDocument component: renders LETTER-size PDF with title "Real Estate Purchase Agreement", property details section (address, city, county, parcel), parties section (seller name, buyer/wholesaler name), financial terms (purchase price, earnest money), each clause from the clauses array, signature lines at bottom, and an audit trail page (only rendered when signers have signedAt data). Follow the code example from 13-RESEARCH.md.
       - AssignmentDocument component: similar but titled "Assignment of Contract", includes original purchase price, assignment fee, and total buyer price (purchasePrice + assignmentFee).
       - Export a generateContractPdf(contract: ContractWithSigners) function that picks the right component based on contractType and calls renderToBuffer. Also export hashBuffer(buffer: Buffer) using crypto.createHash('sha256').
