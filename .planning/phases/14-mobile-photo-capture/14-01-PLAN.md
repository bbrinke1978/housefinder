<task type="auto">
  <name>Task 1: Database schema and blob storage functions</name>
  <files>app/src/db/schema.ts, app/src/lib/blob-storage.ts</files>
  <action>
1. In `app/src/db/schema.ts`, add a `photoCategory` pgEnum with values: "exterior", "kitchen", "bathroom", "living", "bedroom", "garage", "roof", "foundation", "yard", "other".

2. Add a `propertyPhotos` pgTable with columns:
   - `id`: uuid, defaultRandom, primaryKey
   - `dealId`: uuid, nullable, references deals.id
   - `propertyId`: uuid, nullable, references properties.id
   - `isInbox`: boolean, notNull, default false
   - `blobName`: text, notNull (the blob path in Azure Storage)
   - `blobUrl`: text, notNull (internal URL, NOT SAS)
   - `category`: photoCategory, notNull, default "other"
   - `caption`: text, nullable
   - `isCover`: boolean, notNull, default false
   - `sortOrder`: integer, notNull, default 0
   - `fileSizeBytes`: integer, nullable
   - `createdAt`: timestamp with timezone, notNull, defaultNow
   - `updatedAt`: timestamp with timezone, notNull, defaultNow

3. Add indexes: `idx_property_photos_deal_id` on dealId, `idx_property_photos_property_id` on propertyId, `idx_property_photos_is_inbox` on isInbox.

4. Export `PropertyPhotoRow` as `InferSelectModel<typeof propertyPhotos>`.

5. In `app/src/lib/blob-storage.ts`, add:
   - `const PHOTOS_CONTAINER = "photos";`
   - `uploadPhotoBlob(buffer: Buffer, blobName: string): Promise<string>` — same pattern as `uploadContract` (createIfNotExists + upload with contentType "image/jpeg")
   - `generatePhotoSasUrl(blobName: string): string` — same pattern as `generateContractSasUrl` but using PHOTOS_CONTAINER
   - `deletePhotoBlob(blobName: string): Promise<void>` — get container client, get block blob client, call `deleteIfExists()`

Do NOT modify existing functions. Only add new ones.

<task type="auto">
  <name>Task 2: Server actions and query functions</name>
  <files>app/src/lib/photo-actions.ts, app/src/lib/photo-queries.ts</files>
  <action>
1. Create `app/src/lib/photo-queries.ts` with these query functions:
   - `getDealPhotos(dealId: string)` — select all photos where dealId matches, ordered by category then sortOrder. For each photo, generate a SAS URL via `generatePhotoSasUrl(row.blobName)` and return as `{ ...row, sasUrl: string }[]`.
   - `getPropertyPhotos(propertyId: string)` — same but filtered by propertyId.
   - `getInboxPhotos()` — select all photos where `isInbox = true`, ordered by createdAt desc. Include SAS URLs.
   - `getDealCoverPhoto(dealId: string)` — select single photo where dealId matches AND isCover = true. Return `{ blobName, sasUrl } | null`.
   - `getInboxCount()` — select count where isInbox = true. Return number.
   - Export a type `PhotoWithSasUrl = PropertyPhotoRow & { sasUrl: string }`.

2. Create `app/src/lib/photo-actions.ts` as "use server" module with these server actions:
   - `uploadPhoto(formData: FormData)` — extracts fields: file (File), dealId (string|null), propertyId (string|null), category (string), caption (string|null). Converts File to Buffer. Builds blobName: `{deals/dealId | properties/propertyId | inbox}/{uuid}-{sanitized filename}.jpg`. Calls `uploadPhotoBlob(buffer, blobName)`. Inserts into `propertyPhotos` table. Sets `isInbox = true` if both dealId and propertyId are null/empty. If category is "exterior" and this is the first exterior photo for the deal (no existing exterior photo with isCover=true), set `isCover = true`. Calls `revalidatePath` for the relevant page.
   - `setPhotoCover(photoId: string, dealId: string)` — wraps in `db.transaction`: first `UPDATE propertyPhotos SET isCover = false WHERE dealId = X`, then `UPDATE propertyPhotos SET isCover = true WHERE id = photoId`. Calls `revalidatePath(/deals/${dealId})`.
   - `deletePhoto(photoId: string)` — fetches the photo row, calls `deletePhotoBlob(row.blobName)`, deletes the DB row. Calls `revalidatePath` for relevant path.
   - `assignPhotosToDeal(photoIds: string[], dealId: string)` — updates all matching photos: set `dealId`, set `isInbox = false`. If the deal has no cover photo and any assigned photo is category "exterior", set the first one as cover via `setPhotoCover`. Calls `revalidatePath` for `/photos/inbox` and `/deals/${dealId}`.
   - `updatePhotoCaption(photoId: string, caption: string)` — updates caption on the photo row. Revalidates.

All actions must check `auth()` session first (import from `@/auth`). Use `revalidatePath` from `next/cache`.
  </action>
  <verify>cd app && npx tsc --noEmit 2>&1 | grep -c "error" (should be 0 or only pre-existing errors); grep -c "use server" src/lib/photo-actions.ts (should be 1)</verify>
  <done>photo-queries.ts exports 6 query functions with SAS URL generation. photo-actions.ts exports 5 server actions (uploadPhoto, setPhotoCover, deletePhoto, assignPhotosToDeal, updatePhotoCaption) with auth checks and revalidation.</done>
</task>
