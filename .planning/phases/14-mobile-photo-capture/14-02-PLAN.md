  <name>Task 1: Install YARL and create photo upload + gallery components</name>
  <files>app/package.json, app/src/components/photo-upload.tsx, app/src/components/photo-gallery.tsx</files>
  <action>
1. Install `yet-another-react-lightbox` in the app directory: `cd app && npm install yet-another-react-lightbox`.

2. Create `app/src/components/photo-upload.tsx` as "use client" component:
   - Props: `{ dealId?: string; propertyId?: string; onUploadComplete?: () => void }`.
   - Internal state: `queue: PhotoUploadState[]` where each item has `{ id: string, file: File, status: "pending"|"uploading"|"done"|"error", previewUrl: string }`, `category: string` (default "other"), `showCategoryPicker: boolean`.
   - Copy the `resizeImage` function from `receipt-upload.tsx` (canvas resize to max 1600px per CONTEXT.md decision, JPEG 0.8 quality). Change max from 1920 to 1600.
   - TWO file inputs for iOS Safari compatibility (per RESEARCH.md Pitfall 5):
     a) "Take Photo" button with `<input type="file" accept="image/*" capture="environment">` (single shot, no `multiple`)
     b) "Add from Library" button with `<input type="file" accept="image/*" multiple>` (no `capture`)
   - Both inputs share the same `handleCapture` which adds files to the queue with status "pending" and creates preview URLs via `URL.createObjectURL`.
   - After files are queued, show a category selector (dropdown or radio group) with the 10 predefined categories: Exterior, Kitchen, Bathroom, Living, Bedroom, Garage, Roof, Foundation, Yard, Other.
   - Optional caption textarea per photo (collapsible, not required).
   - "Upload N Photos" button triggers sequential upload: for each item, set status to "uploading", call `resizeImage(file)`, create FormData with file/dealId/propertyId/category/caption, call `uploadPhoto(formData)` server action, set status to "done" or "error".
   - Show upload progress as status badges per file: spinner for uploading, checkmark for done, X for error.
   - After all uploads complete, clear queue, revoke all object URLs, call `onUploadComplete?.()`.
   - Use lucide-react icons: Camera, ImagePlus, Check, X, Upload, Loader2.
   - Style with Tailwind: responsive, dark-mode compatible, matching existing zinc/violet palette.

3. Create `app/src/components/photo-gallery.tsx` as "use client" component:
   - Props: `{ photos: PhotoWithSasUrl[]; dealId?: string; canManage?: boolean }` where canManage enables delete/cover/caption controls.
   - Import YARL lightbox via `next/dynamic` with `ssr: false`. Import CSS: `yet-another-react-lightbox/styles.css` and `yet-another-react-lightbox/plugins/captions.css`.
   - Dynamic import the Captions plugin: `dynamic(() => import("yet-another-react-lightbox/plugins/captions").then(m => m.Captions), { ssr: false })`.
   - Group photos by category. For each category group, render a section header (capitalized category name) and a responsive grid (2 cols on mobile, 3 on sm, 4 on md).
   - Each photo card: aspect-square with `object-cover`, rounded corners. If `isCover`, show a small "Cover" badge. Click opens lightbox at that index.
   - Build slides array from all photos: `{ src: photo.sasUrl, title: category, description: caption }`.
   - Lightbox: open state controlled by `index >= 0`, close sets index to -1. Plugins: [Captions].
   - If canManage, each photo card shows overlay controls on hover/tap:
     a) Star icon to set as cover (calls `setPhotoCover(photo.id, dealId)`)
     b) Trash icon to delete (calls `deletePhoto(photo.id)` with confirm dialog)
     c) Caption edit (inline text input, calls `updatePhotoCaption(photo.id, newCaption)`)
   - Empty state: "No photos yet. Add photos to get started." with camera icon.
  </action>
