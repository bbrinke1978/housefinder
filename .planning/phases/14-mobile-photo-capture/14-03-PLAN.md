1. Create `app/src/components/photo-fab.tsx` as "use client" component:
   - Renders a fixed-position circular button (h-14 w-14, rounded-full, bg-primary text-primary-foreground) with Camera icon from lucide-react.
   - Position: `bottom: calc(56px + env(safe-area-inset-bottom, 0px) + 16px)`, `right: 1rem`, z-40.
   - Visible only on mobile: use `md:hidden` class (simpler than useSidebar check — hidden on desktop via CSS).

2. Create `app/src/components/photo-inbox.tsx` as "use client" component:
   - Props: `{ photos: PhotoWithSasUrl[]; deals: { id: string; address: string }[] }`.
   - Renders a grid of inbox photos (same responsive grid as photo-gallery: 2/3/4 cols).
   - Each photo card has a checkbox for multi-select.
   - Above the grid: "Assign to Deal" dropdown (select element listing deals by address) + "Assign" button. Disabled when no photos selected or no deal chosen.
   - On assign: calls `assignPhotosToDeal(selectedPhotoIds, dealId)` server action. Clears selection after success.
   - Category can be set during assignment: show a category dropdown alongside deal selector.
   - Each photo also has a delete button (X icon) that calls `deletePhoto(photo.id)`.
   - Empty state: "No photos in inbox. Use the camera button on mobile to capture photos in the field."

3. Create `app/src/app/(dashboard)/photos/inbox/page.tsx`:
   - Server component that fetches `getInboxPhotos()` and a simple deals list (select id, address from deals table ordered by createdAt desc, limit 50).

   - After the deal INSERT succeeds (after `const inserted = ...`), if `formData.get("propertyId")` is a non-empty string, run a Drizzle UPDATE:
     ```typescript
     await db.update(propertyPhotos)
       .set({ dealId: inserted.id, isInbox: false })
       .where(and(
         eq(propertyPhotos.propertyId, propertyId),
         isNull(propertyPhotos.dealId)
       ));
     ```
   - Import `propertyPhotos` from schema, `and`, `eq`, `isNull` from drizzle-orm.
   - This carries photos from property to the new deal. Non-transactional (best-effort) — if it fails, the deal is still created. Wrap in try/catch with console.error.
