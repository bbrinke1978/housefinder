1. Modify `app/src/lib/deal-actions.ts` (the `createDeal` function):
   - After the deal INSERT succeeds (after `const inserted = ...`), if `formData.get("propertyId")` is a non-empty string, run a Drizzle UPDATE:
     ```typescript
     await db.update(propertyPhotos)
       .set({ dealId: inserted.id, isInbox: false })
       .where(and(
         eq(propertyPhotos.propertyId, propertyId),
         isNull(propertyPhotos.dealId)
       ));
     ```
   - Import `propertyPhotos` from schema, `and`, `eq`, `isNull` from drizzle-orm.
   - This carries photos from property to the new deal. Non-transactional (best-effort) -- if it fails, the deal is still created. Wrap in try/catch with console.error.
   - Also: if the deal now has photos but no cover, find the first "exterior" photo and set it as cover (simple UPDATE where category='exterior' ORDER BY createdAt LIMIT 1, set isCover=true). This is a nice-to-have; if it complicates things, skip it -- the user can set cover manually.
