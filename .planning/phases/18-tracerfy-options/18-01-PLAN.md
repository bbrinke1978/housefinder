Create `app/src/lib/tracerfy-actions.ts` as a `"use server"` module. Port and adapt logic from `scraper/src/sources/tracerfy-enrichment.ts` with these CRITICAL fixes and adaptations:

**CRITICAL BUG FIX:** Use correct PascalCase-dash field names from Tracerfy API responses:
- Emails: `Email-1` through `Email-5` (NOT `email_1`)
- Mobiles: `Mobile-1` through `Mobile-5` (NOT `mobile_1`)
- Landlines: `Landline-1` through `Landline-3` (NOT `landline_1`)
- `primary_phone` is correct (lowercase)

**Imports:**
- `db` from `@/db/client`
- `properties, ownerContacts, scraperConfig` from `@/db/schema`
- `eq, and, inArray, isNull, sql, like` from `drizzle-orm`
- `auth` from `@/auth`
- `z` from `zod/v4`
- `revalidatePath` from `next/cache`
- Types from `@/types/index`

**Constants:**
- `BASE_URL = "https://tracerfy.com/v1/api"`
- `POLL_INTERVAL_MS = 3000` (faster than scraper's 5000 ŌĆö better UX)
- `MAX_POLL_MS = 25000` (stay under Netlify 26s limit)
- `COST_PER_TRACE = 0.02` (for cost estimation display)

**Helper functions (private):**
1. `parseOwnerName(name: string)` ŌĆö copy from scraper, returns `{ first_name, last_name }`
2. `tracerfyFetch(method, path, apiKey, body?)` ŌĆö adapted from scraper, handles auth header and JSON
3. `submitBatch(props, apiKey)` ŌĆö submit to POST /trace/ with json_data and column mappings. Include `property_id` in json_data for result matching. Return queue_id.
4. `pollUntilComplete(queueId, apiKey, maxWaitMs)` ŌĆö poll GET /queues/ every 3s until pending=false and download_url exists. Throw on timeout.
5. `fetchQueueResults(queueId, apiKey)` ŌĆö GET /queue/:id, returns result rows
6. `extractPhonesAndEmails(row)` ŌĆö extract using CORRECT PascalCase-dash field names. Returns `{ phones: Array<{ number: string, type: "mobile" | "landline" | "unknown" }>, emails: string[] }`. Determine type from field name: `Mobile-N` ŌåÆ "mobile", `Landline-N` ŌåÆ "landline", `primary_phone` ŌåÆ "unknown".
7. `storeResults(results, props)` ŌĆö store in ownerContacts with source="tracerfy" for primary phone, onConflictDoUpdate on [propertyId, source]. Store additional phones with type-encoded source: "tracerfy-mobile-2", "tracerfy-landline-1" etc (format: `tracerfy-{type}-{N}`). Store additional emails as "tracerfy-2", "tracerfy-3" etc. Store mailing address as "tracerfy-address" source. Store "not found" marker (phone=null, email=null) for properties with no results so they won't be re-queried.
8. `recordRun(entry: TracerfyRunEntry)` ŌĆö append to scraperConfig key `tracerfy.runHistory` (JSON array, keep last 50). Also update `tracerfy.monthlySpend` JSON for current month.

**Exported server actions:**

1. `runSkipTrace(propertyId: string)` ŌĆö auth check, get API key from env, fetch property data from DB (address, city, state, zip, ownerName), submit single-item batch, poll, store results, record run, revalidatePath for property detail. Returns `{ success: true, found: boolean, phoneCount: number, emailCount: number }` or `{ error: string }`.

2. `runBulkSkipTrace(propertyIds: string[])` ŌĆö auth check, get API key, fetch all properties in one query using inArray, filter to those with valid addresses, submit batch, poll, store results, record run, revalidatePath("/"). Returns `{ success: true, found: number, notFound: number, creditsUsed: number }` or `{ error: string }`.

3. `getTracerfyStatus()` ŌĆö check if TRACERFY_API_KEY env var exists. If yes, call GET /analytics/ to get balance. Returns `TracerfyStatus` type. No auth check needed (settings page).

4. `getTracerfyRunHistory()` ŌĆö read scraperConfig key `tracerfy.runHistory`, parse JSON, return `TracerfyRunEntry[]`. Returns empty array if key not found.

5. `getTracerfyConfig()` ŌĆö read lowBalanceThreshold and monthlyCap from scraperConfig. Return `TracerfyConfig` with defaults.

6. `saveTracerfyConfig(config: { lowBalanceThreshold: number, monthlyCap: number })` ŌĆö validate with zod, save to scraperConfig keys. Returns `{ success: true }` or `{ error: string }`.

**Key patterns to follow:**
- Use `useTransition` compatible return types (union of success/error, not throw)
- Use `onConflictDoUpdate` target `[ownerContacts.propertyId, ownerContacts.source]` for upserts
- Use `scraperConfig` key-value pattern (like mail-settings-actions.ts) for config storage
- Call `revalidatePath` after mutations to refresh UI
