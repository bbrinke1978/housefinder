**1. Create `skip-trace-confirm-dialog.tsx`** — a shared "use client" dialog used by both single and bulk triggers.

Props:
- `open: boolean`
- `onOpenChange: (open: boolean) => void`
- `propertyCount: number`
- `estimatedCost: number` (in dollars, = propertyCount * 0.02)
- `currentBalance: number | null` (fetched live)
- `lowBalanceThreshold: number` (from config, default 2.00)
- `onConfirm: () => void`
- `isPending: boolean`
- `balanceLoading: boolean`

Use `@base-ui/react/dialog` (NOT Radix/shadcn Dialog). Pattern: `Dialog.Root open/onOpenChange`, `Dialog.Portal`, `Dialog.Backdrop`, `Dialog.Popup`.

Content:
- Title: "Confirm Skip Trace"
- Body: "Skip trace {propertyCount} propert{y|ies}? Estimated cost: ${estimatedCost.toFixed(2)}"
- Balance line: "Current balance: ${balance.toFixed(2)}" or "Balance unavailable" if null, with a loading spinner if balanceLoading
- Low balance warning: yellow banner when balance !== null && balance < lowBalanceThreshold: "Low balance — consider topping up your Tracerfy account"
- Cancel button: `Dialog.Close` directly with className (NO asChild — @base-ui/react does not support it)
- Confirm button: calls onConfirm, disabled when isPending, shows "Tracing..." when pending

Style: Use project Tailwind tokens (bg-background, text-foreground, border, rounded-2xl, etc.). Backdrop with bg-black/50 fixed inset-0.

**2. Create `skip-trace-button.tsx`** — "use client" component for the ContactTab.

Props:
- `propertyId: string`
- `hasTracerfyResult: boolean` (true if any contact has source starting with "tracerfy")
- `onTraceComplete?: () => void`

Behavior:
- If `hasTracerfyResult` is true, show a disabled Badge: "Skip traced" with a green check icon
- If false, show a Button "Skip Trace" with a Search icon
- On click: fetch live balance via `getTracerfyStatus()` and config via `getTracerfyConfig()`, then open SkipTraceConfirmDialog with count=1
- On confirm: call `runSkipTrace(propertyId)` via useTransition
- On success with found=true: show success message "Found {phoneCount} phone(s), {emailCount} email(s)" (the page will revalidate automatically from the server action)
- On success with found=false: show "No results found" message
- On error: show error text in red
