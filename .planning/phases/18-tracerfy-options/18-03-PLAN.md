**2. Create `app/src/components/skip-tracing-settings.tsx`** — "use client" component.

Props: `{ status: TracerfyStatus, runHistory: TracerfyRunEntry[], config: TracerfyConfig }`

Sections (each in a Card):

**A. Connection Status Card:**
- Green circle + "Connected" if `status.configured && !status.error`
- Red circle + "Not Configured" if `!status.configured` with message "Set TRACERFY_API_KEY in your Netlify environment variables"
- Yellow circle + "Error" if configured but has error, showing error message
- Balance display: large number "${status.balance?.toFixed(2)}" or "—" if null
- Low balance warning banner (yellow bg): show when balance !== null && balance < config.lowBalanceThreshold

**B. Monthly Spend Card:**
- Parse the most recent month from run history entries (filter by current month YYYY-MM from date field)
- Show total credits used this month
- Show progress bar: `(monthlySpend / config.monthlyCap) * 100`%
- If monthlySpend > monthlyCap: orange warning "Monthly soft cap exceeded ($X.XX / $Y.YY) — this is a warning, not a hard block"
- Show the month name (e.g., "April 2026")

**C. Run History Table Card:**
- Title: "Recent Runs"
- Table with columns: Date, Properties, Found, Not Found, Credits
- Show last 20 entries from runHistory (most recent first)
- Date formatted with toLocaleDateString("en-US", { timeZone: "America/Denver" })
- Found rate as percentage in a subtle badge
- If no history: "No skip trace runs yet"

**D. Configuration Card:**
- Title: "Cost Controls"
- Form with two inputs:
  - "Low Balance Warning Threshold ($)" — number input, default from config.lowBalanceThreshold
  - "Monthly Soft Cap ($)" — number input, default from config.monthlyCap
- Save button that calls `saveTracerfyConfig()` via useTransition
- Success/error feedback inline

**3. Add sidebar link** — In `sidebar.tsx`, add a "Skip Tracing" link under Settings section (alongside existing Mail Settings link). Use `Search` icon from lucide-react. Link to `/settings/skip-tracing`.

**If `hasContacts` is false and the form has a propertyId prefill:**
- Before the form renders (or as a modal on mount), show a dialog:
  - Title: "No Contact Info"

  - Cost estimate: "Estimated cost: ~$0.02"
  - "Skip Trace First" button: calls `runSkipTrace(propertyId)` via useTransition, shows loading state, then on success dismisses dialog and continues to form
  - "Skip" button: dismisses dialog, continues to form without tracing
- Use `@base-ui/react/dialog` for the dialog
- Show the dialog once on mount via a `useState<boolean>(true)` that defaults to true when hasContacts is false
