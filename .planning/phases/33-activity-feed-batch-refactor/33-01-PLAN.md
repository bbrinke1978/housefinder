   - Use `const` everywhere. No `as any`. No `let`. (Pitfall §5 in RESEARCH.md — prefer-const is an ERROR-level rule in this project; commit 07c407b proved it.)

3. CRITICAL pitfall checks before declaring done (all from RESEARCH.md §"Common Pitfalls"):
   - Pitfall §1: audit_log JOIN is split into TWO UNION ALL legs — one with `entity_type = 'lead' AND entity_id = leads.id`, one with `entity_type = 'deal' AND entity_id = deals.id`. Both legs include `entity_type IN ('lead'|'deal')` so the composite index `idx_audit_log_entity` is used (Pitfall §6).

   - Pitfall §3: must use `ROW_NUMBER()` window — DO NOT use `DISTINCT ON` (incompatible with `COUNT(*) OVER` in same SELECT).
