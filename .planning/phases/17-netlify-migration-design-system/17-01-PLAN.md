  <files>app/netlify.toml, app/next.config.ts</files>
  <action>
  1. Create `app/netlify.toml` with the exact nobshomes pattern:
     ```toml
     [build]
       command = "npm run build"
       publish = ".next"

     [build.environment]
       NODE_VERSION = "20"
     ```
     Do NOT add a `[[plugins]]` block — Netlify auto-installs @netlify/plugin-nextjs for Next.js projects.

  2. Update `app/next.config.ts`:
     - Remove `output: "standalone"` (standalone is for Azure/Docker; OpenNext requires default output)
     - Add `images.remotePatterns` for Azure Blob Storage to prevent photo breakage after migration:
       ```typescript
       images: {
         remotePatterns: [
           { protocol: "https" as const, hostname: "*.blob.core.windows.net" },
         ],
       },
       ```
     - Keep existing `transpilePackages: ["mapbox-gl"]` and `serverExternalPackages: ["@react-pdf/renderer"]`
  </action>
  <verify>
    <automated>cd app && cat netlify.toml && grep -v "standalone" next.config.ts && grep "remotePatterns" next.config.ts</automated>
  </verify>
  <done>netlify.toml exists with build command and Node 20; next.config.ts has no standalone output and includes blob storage remotePatterns</done>
</task>
