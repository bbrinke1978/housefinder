<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Brian sets up Netlify site and environment variables</name>
  <what-to-do>
  Brian needs to:
  1. Go to Netlify Dashboard (https://app.netlify.app)
  2. Add new site -> Import from Git -> GitHub -> select `housefinder` repo
  3. Set base directory to `app` in build settings
  4. Build command: `npm run build` (should auto-detect from netlify.toml)
  5. Publish directory: `.next` (should auto-detect from netlify.toml)
  6. Add ALL environment variables (listed in user_setup above):
     - AUTH_SECRET: Generate new one with `openssl rand -base64 33`
     - AUTH_EMAIL: Same as Azure App Service
     - AUTH_PASSWORD_HASH: Same as Azure App Service
     - DATABASE_URL: Same as Azure (port 5432, NOT 6432)
     - NEXTAUTH_URL: `https://<your-site>.netlify.app`
     - NEXT_PUBLIC_APP_URL: Same as NEXTAUTH_URL
     - NEXT_PUBLIC_MAPBOX_ACCESS_TOKEN: Same as Azure
     - RESEND_API_KEY: Same as Azure
     - AZURE_STORAGE_CONNECTION_STRING: Same as Azure
     - AZURE_DOCUMENT_INTELLIGENCE_ENDPOINT: Same as Azure
     - AZURE_DOCUMENT_INTELLIGENCE_KEY: Same as Azure
     - WEBSITE_LEAD_API_KEY: Same as Azure
  7. Trigger first deploy (push to master or manual deploy trigger)
  8. Wait for build to succeed

  **NOTE on PgBouncer:** Azure PgBouncer is NOT supported on B1ms Burstable tier. Keep DATABASE_URL on port 5432 (standard PostgreSQL). This is documented as a future upgrade item, not a blocker.

  **NOTE on Azure App Service:** Keep Azure App Service running as fallback for ~1 week after successful Netlify migration. Then Brian can shut it down.
  </what-to-do>
