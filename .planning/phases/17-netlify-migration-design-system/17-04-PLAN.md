<task type="auto">
  <name>Task 1: Install framer-motion and create SwipeCard component</name>
  <files>app/package.json, app/src/components/swipe-card.tsx</files>
  <action>
  1. Install framer-motion:
     ```bash
     cd app && npm install framer-motion
     ```

  2. Create `app/src/components/swipe-card.tsx` — a reusable "use client" component:

     ```typescript
     "use client";
     import { motion, useMotionValue, useTransform, animate } from "framer-motion";
     import { Phone, ChevronRight } from "lucide-react";

     interface SwipeCardProps {
       children: React.ReactNode;
       onSwipeLeft?: () => void;   // Status change action
       onSwipeRight?: () => void;  // Call action
       leftLabel?: string;         // Default: "Status"
       rightLabel?: string;        // Default: "Call"
       disabled?: boolean;         // Disable swipe (e.g., desktop)
     }
     ```

     Implementation:
     - Use `motion.div` with `drag="x"` and `dragDirectionLock` to prevent vertical scroll conflicts
     - Set `overscrollBehaviorX: "contain"` on the outer container via style prop
     - SWIPE_THRESHOLD = 80px
     - On drag end: if offset.x < -SWIPE_THRESHOLD, animate to -120px and call onSwipeLeft; if > SWIPE_THRESHOLD, animate to 120px and call onSwipeRight; else snap back to 0
     - Left action zone (revealed on right swipe): green background with Phone icon + rightLabel
     - Right action zone (revealed on left swipe): brand-500 background with ChevronRight icon + leftLabel
     - After action callback fires, animate back to x=0 after 300ms delay
     - Use `dragConstraints={{ left: -150, right: 150 }}` and `dragElastic={0.3}`
     - Apply `touch-action: pan-y` via className on the motion.div to allow vertical scroll to pass through
     - Component renders as `<div className="relative overflow-hidden rounded-2xl">` wrapper

  3. Integrate SwipeCard into property cards on the dashboard. Find the component that renders property cards in the dashboard list (likely in `app/src/app/(dashboard)/page.tsx` or a PropertyCard component). Wrap each card's content with `<SwipeCard>` on mobile only:
     - Use a `md:hidden` wrapper for swipe-enabled cards on mobile
     - Use the regular card for `hidden md:block` on desktop
     - onSwipeRight: if owner phone exists, trigger `window.location.href = 'tel:' + phone`
     - onSwipeLeft: navigate to property detail page (simple approach — or cycle status if feasible)

  If there is a dedicated `PropertyCard` or similar reusable component, integrate there. If cards are rendered inline in page.tsx, create a thin wrapper.

  **Important:** Use `dragDirectionLock` to prevent the framer-motion horizontal drag from capturing vertical scroll gestures. This is critical for mobile UX — without it, the card list becomes unscrollable.
  </action>
