**blob-storage.ts:**
Create module with two functions:

1. `uploadBlob(buffer: Buffer, blobName: string, contentType: string): Promise<string>` — Upload to Azure Blob Storage container "receipts". Use BlobServiceClient from connection string (env: AZURE_STORAGE_CONNECTION_STRING). Return the blob URL. Container is private access level.

2. `generateSasUrl(blobName: string): string` — Generate a SAS URL with 1-hour read-only expiry for displaying receipt images in the browser. Use generateBlobSASQueryParameters with BlobSASPermissions.parse("r").

Blob name format: `${budgetId}/${uuid()}-${originalFilename}` for organization.

**ocr.ts:**
Create module with one function:

1. `analyzeReceipt(blobUrl: string): Promise<{ vendor: string | null; date: string | null; totalCents: number | null }>` — Call Azure Document Intelligence prebuilt-receipt model. Use DocumentAnalysisClient with AzureKeyCredential (env: AZURE_DOCUMENT_INTELLIGENCE_ENDPOINT, AZURE_DOCUMENT_INTELLIGENCE_KEY). Extract MerchantName, TransactionDate, Total from the first receipt in the response. Convert Total (float dollars) to integer cents. Return nulls if any field not detected (graceful degradation per research pitfall 5).

Wrap the entire OCR call in try/catch. On any error, log the error and return all nulls — never block expense creation on OCR failure.

**receipt-actions.ts:**
"use server" module. One main server action:

1. `uploadReceipt(formData: FormData): Promise<{ receiptId: string; vendor: string | null; date: string | null; totalCents: number | null }>` — Extract: file (File object from formData), budgetId.

Steps:
a. Read file as ArrayBuffer, convert to Buffer
b. Client-side resize is handled by the component — server receives the already-resized image
c. Generate blobName: `${budgetId}/${crypto.randomUUID()}-${file.name}`
d. Upload to blob storage via uploadBlob
e. Call analyzeReceipt with the blob URL
f. Insert receipt row into DB (blobUrl, blobName, ocrRawJson if available, vendor, receiptDate, totalCents)
g. Return receipt ID + OCR-extracted fields for pre-filling expense form
h. Revalidate the deal page path

Use .js extensions in all imports (ESM requirement).

**ReceiptUpload component (receipt-upload.tsx):**
"use client" component. Props: budgetId (string), categories (BudgetCategory[]), dealId (string), onExpenseCreated (callback to refresh).

UI flow:
1. Large "Scan Receipt" button (Camera icon from lucide-react). Uses hidden `<input type="file" accept="image/*" capture="environment">` triggered by button click — native camera on mobile, file picker on desktop (per research section 4).

2. On file selection:
   a. Client-side resize using canvas: max 1920px on longest side, JPEG at 0.8 quality (reduces ~5MB phone photo to ~400KB per research pitfall 3). Create a helper function `resizeImage(file: File): Promise<Blob>`.
   b. Show loading spinner with "Scanning receipt..."
   c. Submit resized image to uploadReceipt server action via FormData
   d. On response, show pre-filled expense form:
      - Vendor: pre-filled from OCR (editable)
      - Amount: pre-filled from OCR totalCents (editable, shown as dollars)
      - Date: pre-filled from OCR date (editable)
      - Category: dropdown (user selects — OCR doesn't reliably detect this)
      - "Save Expense" button to call addExpense with the receiptId linked

3. If OCR returns all nulls: show the expense form with empty fields and a note "Could not auto-detect receipt details — please enter manually."

4. Show thumbnail of the uploaded image next to the form for reference.

**ReceiptGallery component (receipt-gallery.tsx):**
"use client" component. Props: receipts (array of receipt rows with SAS URLs).

Display receipt thumbnails in a grid (2 cols on mobile, 4 on desktop). Each thumbnail shows:
- Receipt image (use `<img>` with SAS URL)
- Vendor name (if known)
- Date (if known)
- Total (if known)

Clicking a thumbnail opens the full image in a new tab (simple `<a target="_blank">`). No modal/lightbox — keep it simple.

Note: SAS URLs must be generated server-side. The parent component (budget-tab.tsx) should call generateSasUrl for each receipt and pass the results down.
