Add 4 new tables to schema.ts after the dealNotes table, following the exact schema from 09-RESEARCH.md section 6:

1. `budgets` table: id (uuid PK), dealId (uuid FK to deals.id, unique — one budget per deal), totalPlannedCents (integer, default 0), contingencyCents (integer, default 0 — auto-calculated as 10% of totalPlannedCents per user decision), notes (text nullable), createdAt, updatedAt.

2. `budget_categories` table: id (uuid PK), budgetId (uuid FK to budgets.id), name (text), sortOrder (integer), plannedCents (integer, default 0), createdAt. Indexes: idx_budget_categories_budget_id on budgetId, uniqueIndex uq_budget_category_name on (budgetId, name).

3. `receipts` table: id (uuid PK), budgetId (uuid FK to budgets.id), blobUrl (text), blobName (text), ocrRawJson (text nullable), vendor (text nullable), receiptDate (date nullable), totalCents (integer nullable), isDeleted (boolean default false), createdAt.

4. `expenses` table: id (uuid PK), budgetId (uuid FK to budgets.id), categoryId (uuid FK to budget_categories.id), receiptId (uuid FK to receipts.id, nullable), vendor (text nullable), description (text nullable), amountCents (integer), expenseDate (date), notes (text nullable), createdAt, updatedAt. Indexes: idx_expenses_budget_id, idx_expenses_category_id, idx_expenses_receipt_id.

Export InferSelectModel types: BudgetRow, BudgetCategoryRow, ReceiptRow, ExpenseRow.

In types/index.ts, add:

```typescript
export const DEFAULT_BUDGET_CATEGORIES = [
  "Demo / Site Prep", "Foundation / Structural", "Framing / Carpentry",
  "Roofing", "Exterior / Siding", "Windows / Doors", "Plumbing",
  "Electrical", "HVAC", "Insulation", "Drywall", "Paint / Finish Work",
  "Flooring", "Kitchen", "Bathrooms", "Interior Trim", "Landscaping",
  "Permits / Fees", "Miscellaneous"
] as const;
// Note: "Contingency" is NOT a category — it's a separate 10% reserve on the budget level

export interface BudgetCategory {
  id: string;
  name: string;
  sortOrder: number;
  plannedCents: number;
  actualCents: number; // computed from expenses SUM
}

export interface BudgetSummary {
  id: string;
  dealId: string;
  totalPlannedCents: number;
  contingencyCents: number; // 10% of totalPlannedCents, auto-calculated
  totalSpentCents: number; // SUM of all expenses
  remainingCents: number; // totalPlannedCents + contingencyCents - totalSpentCents
  percentUsed: number; // totalSpentCents / (totalPlannedCents + contingencyCents) * 100
  categories: BudgetCategory[];
  notes: string | null;
  createdAt: Date;
  updatedAt: Date;
}

export interface ExpenseRow {
  id: string;
  budgetId: string;
  categoryId: string;
  categoryName: string; // from join
  receiptId: string | null;
  vendor: string | null;
  description: string | null;
  amountCents: number;
  expenseDate: string;
  notes: string | null;
  createdAt: Date;
}

// Budget health status for MAO profit indicators (per user decision)
export type BudgetHealth = "profitable" | "break_even" | "loss";
```

<task type="auto">
  <name>Task 2: Budget queries and server actions</name>
  <files>app/src/lib/budget-queries.ts, app/src/lib/budget-actions.ts</files>
  <action>
Create budget-queries.ts with these functions:

1. `getBudgetByDealId(dealId: string): Promise<BudgetSummary | null>` — Fetch budget + categories for a deal. Use LEFT JOIN on expenses to compute actualCents per category via SUM(amount_cents). Compute totalSpentCents, remainingCents, percentUsed in JS. Contingency = 10% of totalPlannedCents (read from budget row). Return null if no budget exists for this deal.

2. `getExpenses(budgetId: string): Promise<ExpenseRow[]>` — Fetch all expenses for a budget, JOIN budget_categories to get categoryName. Order by expenseDate DESC.

3. `getReceipts(budgetId: string)` — Fetch non-deleted receipts for a budget. Order by createdAt DESC.

Create budget-actions.ts with these server actions (use "use server" directive, revalidatePath pattern from deal-actions.ts):

1. `createBudget(dealId: string): Promise<{ id: string }>` — Create budget row with totalPlannedCents = deal.repairEstimate (auto-populate per user decision). Compute contingencyCents = Math.round(totalPlannedCents * 0.10). Seed DEFAULT_BUDGET_CATEGORIES as budget_categories rows with sortOrder matching array index and plannedCents = 0. Revalidate `/deals/${dealId}`.

2. `updateCategoryPlanned(categoryId: string, plannedCents: number, dealId: string)` — Update plannedCents for a category. Also recalculate and update budget.totalPlannedCents as SUM of all categories, and contingencyCents as 10% of that total. Revalidate path.

3. `addExpense(formData: FormData)` — Extract budgetId, categoryId, vendor, description, amountCents, expenseDate, notes, receiptId (optional). Insert expense row. Revalidate path.

4. `deleteExpense(expenseId: string, dealId: string)` — Delete expense row. Revalidate path.

5. `updateBudgetNotes(budgetId: string, notes: string, dealId: string)` — Update budget notes. Revalidate path.

Follow existing patterns: use db from "@/db/index", .js extensions in imports (ESM), revalidatePath from "next/cache".
