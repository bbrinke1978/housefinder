**BudgetAlertBanner component (budget-alerts.tsx):**
"use client" component. Props: percentUsed (number), remainingCents (number), contingencyCents (number), totalPlannedCents (number), totalSpentCents (number).

Logic:
- If percentUsed < 80: no banner (return null)
- If 80 <= percentUsed < 100: yellow banner "Approaching budget - ${formatDollars(remainingCents)} remaining"
- If percentUsed >= 100 and totalSpentCents <= totalPlannedCents + contingencyCents: orange banner "Over planned budget - eating into contingency reserve"
- If totalSpentCents > totalPlannedCents + contingencyCents: red banner "OVER BUDGET by ${formatDollars(totalSpentCents - totalPlannedCents - contingencyCents)} - review expenses"

Use Tailwind: bg-yellow-50 border-yellow-400, bg-orange-50 border-orange-400, bg-red-50 border-red-400 with appropriate text colors. Include AlertTriangle icon from lucide-react.

**BudgetCharts component (budget-charts.tsx):**
"use client" component. Props: categories (BudgetCategory[]).

Two charts (same recharts patterns proven in Phase 6):

1. **Category Progress Bars** (not a recharts chart — plain HTML/Tailwind):
   For each category with plannedCents > 0 OR actualCents > 0:
   - Category name (left)
   - Progress bar: width = min(100, (actualCents / plannedCents) * 100)%
   - Color: green (< 80%), yellow (80-99%), red (>= 100%)
   - Label: "$X / $Y (Z%)" on the right

   If plannedCents is 0 and actualCents > 0: show red at 100% width with "Unplanned: $X"

   Use a div with dynamic width and bg-color. Pattern from research:
   ```
   Demo         [========--------] $4,200 / $5,000  (84% - YELLOW)
   Plumbing     [================] $8,100 / $7,500  (108% - RED)
   ```

2. **Pie Chart — Spending by Category** (Recharts PieChart):
   Data: categories with actualCents > 0 only (skip zero-spend categories).
   Use PieChart > Pie > Cell with a color array. Include Tooltip with custom formatter showing dollar amounts. Include Legend. Use the same recharts import pattern from Phase 6 (dynamic import to avoid SSR issues if needed).

3. **Grouped Bar Chart — Planned vs Actual** (Recharts BarChart):
   Data: categories with plannedCents > 0 OR actualCents > 0.
   Two bars per category: gray for planned, blue for actual (red if actual > planned).
   Use layout="vertical" for horizontal bars (category names on Y axis — same pattern as Phase 6 attribution chart).
   Include Tooltip showing dollar amounts.

Wrap charts in a collapsible section on mobile: "Charts" heading with expand/collapse (use simple state toggle, not accordion). Charts are secondary to progress bars per research recommendation.
