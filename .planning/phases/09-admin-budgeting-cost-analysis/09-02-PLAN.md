**BudgetTab component (budget-tab.tsx):**
"use client" component. Props: deal (DealWithBuyer), budget (BudgetSummary | null).

If no budget exists: Show a "Create Budget" button. On click, call createBudget(deal.id) server action. Display what repair_estimate will auto-populate as (format as dollars: `$${(deal.repairEstimate ?? 0) / 100}`).

If budget exists, show:

1. **KPI Header** — single row on mobile:
   - Total Budget: totalPlannedCents + contingencyCents (formatted as dollars)
   - Spent: totalSpentCents
   - Remaining: remainingCents
   - % Used: percentUsed (colored: green < 80%, yellow 80-99%, red >= 100%)

2. **Profit Indicator** (per user decision — MAO math):
   - Compute: profitGap = deal.repairEstimate - totalSpentCents
   - If totalSpentCents < deal.repairEstimate: green badge "Profitable — $X under estimate"
   - If totalSpentCents === deal.repairEstimate: yellow badge "Break Even"
   - If totalSpentCents > deal.repairEstimate: red badge "Loss — $X over estimate"
   - Only show if deal.repairEstimate is set

3. **Contingency Reserve** — show "10% Contingency: $X" as a separate line item below the KPI header. If spending exceeds planned total (excl contingency), show yellow "Eating into contingency" warning.

4. **Category list** — render BudgetCategoryEditor for each category.

5. **Expense section** — "Add Expense" button + ExpenseList below.

**BudgetCategoryEditor component (budget-category-editor.tsx):**
"use client" component. Props: category (BudgetCategory), dealId (string).

Display each category as a row:
- Category name (left)
- Planned: editable input (dollar amount, convert to/from cents). On blur, call updateCategoryPlanned server action.
- Actual: display actualCents formatted as dollars (read-only)
- Variance: planned - actual (green if positive, red if negative)

Categories are in fixed order per user decision (no drag-and-drop). Use sortOrder from DB.

Use the same form patterns as deal-actions.ts — FormData-based server actions with useActionState if needed, or simpler onChange + fetch pattern.
