## Decisions Made

- `analyzeReceipt` wraps entire OCR pipeline in try/catch and returns all nulls on any error — ensures OCR failure never blocks expense creation or upload flow
- SAS URL is generated by parsing AccountName and AccountKey from the single AZURE_STORAGE_CONNECTION_STRING — avoids adding separate credential env vars
- ReceiptGallery uses a plain `<a target="_blank">` link for full image view — no modal/lightbox complexity as specified in plan
- Client-side canvas resize runs before upload: max 1920px on longest side, JPEG quality 0.8, reducing ~5MB phone photos to ~400KB

## User Setup Required

**External services require manual configuration before runtime use:**

### Azure Document Intelligence
1. Create Document Intelligence resource (F0 free tier) in Azure Portal
   - Search "Document Intelligence" -> F0 tier -> rg-housefinder -> West US 3
2. Add env vars to Azure App Service:
   - `AZURE_DOCUMENT_INTELLIGENCE_ENDPOINT` — from Keys and Endpoint -> Endpoint
   - `AZURE_DOCUMENT_INTELLIGENCE_KEY` — from Keys and Endpoint -> KEY 1

### Azure Blob Storage
1. In Azure Portal, navigate to Storage Account -> Containers
2. Create container named `receipts` with Private access level
3. Add env var to Azure App Service:
   - `AZURE_STORAGE_CONNECTION_STRING` — from Storage Account -> Access keys -> Connection string
