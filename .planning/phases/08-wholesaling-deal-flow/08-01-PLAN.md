  <name>Task 1: Add deals/buyers/dealNotes schema + types + migration</name>
  <files>app/src/db/schema.ts, app/src/types.ts</files>
  <action>
Add three new table definitions to schema.ts following existing pgTable conventions (UUID PKs, withTimezone timestamps, text for status fields):

1. `deals` table with fields:
   - id (uuid, defaultRandom, PK)
   - propertyId (uuid, nullable FK to properties.id — NO onDelete cascade)
   - address (text, notNull), city (text, notNull), state (text, notNull, default "UT")
   - sellerName (text), sellerPhone (text)
   - condition (text — "light", "medium", "heavy", "tear_down")
   - timeline (text — "asap", "1_month", "3_months", "flexible")
   - motivation (text — "inherited", "financial_distress", "vacant", "divorce", "other")
   - askingPrice (integer, dollars)
   - arv (integer), repairEstimate (integer), wholesaleFee (integer, default 15000), mao (integer), offerPrice (integer)
   - status (text, notNull, default "lead") — values: lead | qualified | analyzed | offered | under_contract | marketing | assigned | closing | closed | dead
   - assignedBuyerId (uuid, nullable FK to buyers.id)
   - assignmentFee (integer), closingDate (date)
   - contractStatus (text, nullable) — null | sent | signed | in_escrow | title_clear | closing_scheduled
   - earnestMoney (integer, default 100), inspectionDeadline (date), earnestMoneyRefundable (boolean, default true)
   - createdAt, updatedAt (timestamps with timezone, defaultNow)
   - Indexes: idx_deals_status on status, idx_deals_property_id on propertyId

2. `buyers` table with fields:
   - id (uuid, defaultRandom, PK)
   - name (text, notNull), phone (text), email (text)
   - buyBox (text), minPrice (integer), maxPrice (integer)
   - fundingType (text — "cash" | "hard_money" | "both")
   - targetAreas (text), rehabTolerance (text — "light" | "medium" | "heavy" | "any")
   - notes (text), isActive (boolean, notNull, default true)
   - createdAt, updatedAt (timestamps)

3. `dealNotes` table with fields:
   - id (uuid, defaultRandom, PK)
   - dealId (uuid, notNull, FK to deals.id)
   - noteText (text, notNull), noteType (text, notNull, default "user" — "user" | "status_change")
   - previousStatus (text), newStatus (text)
   - createdAt (timestamp)
   - Index: idx_deal_notes_deal_id on dealId

Export all three tables from schema.ts. Add the exports to the existing imports in schema.ts.

In types.ts, add:
- `DEAL_STATUSES` const array with all 10 status values
- `DealStatus` type from the const array
- `DealWithBuyer` type with all deal fields plus optional buyerName/buyerId from join
- `Buyer` type (inferred from buyers table)
- `DealNote` type (inferred from dealNotes table)
- `CONTRACT_STATUSES` const array
- `CONDITION_OPTIONS`, `TIMELINE_OPTIONS`, `MOTIVATION_OPTIONS` const arrays for form dropdowns

Run `npx drizzle-kit generate` from app/ directory to generate migration SQL.

IMPORTANT: Use `text` for status fields, NOT pgEnum (per research — 10 statuses is unwieldy as Postgres enum). Use `zod/v4` enum validation in server actions instead. Do NOT use drizzle `relations()` — follow existing direct join pattern.
  </action>

  </verify>
