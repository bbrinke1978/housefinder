  <files>app/src/app/(dashboard)/deals/buyers/page.tsx, app/src/components/buyer-list.tsx, app/src/components/buyer-intake-form.tsx, app/src/lib/deal-queries.ts, app/src/lib/deal-actions.ts</files>
  <action>
1. Add to deal-queries.ts:
   - `getBuyers()` — SELECT all from buyers WHERE isActive = true, order by name asc. Return Buyer[].
   - `getMatchingBuyers(dealPrice: number)` — SELECT from buyers WHERE isActive = true AND (maxPrice IS NULL OR maxPrice >= dealPrice) AND (minPrice IS NULL OR minPrice <= dealPrice). Return Buyer[].

2. Add to deal-actions.ts:
   - `createBuyer(formData: FormData)` — Parse with zod: name (required), phone, email, buyBox, minPrice, maxPrice, fundingType, targetAreas, rehabTolerance, notes. Insert into buyers. revalidatePath("/deals/buyers").
   - `updateBuyer(buyerId: string, formData: FormData)` — Same fields, UPDATE. revalidatePath.
   - `deactivateBuyer(buyerId: string)` — Set isActive = false. revalidatePath. (Soft delete, not hard delete.)
   - `assignBuyerToDeal(dealId: string, buyerId: string, assignmentFee: number)` — Update deals.assignedBuyerId, deals.assignmentFee, set status to "assigned" if currently "marketing". Insert status_change note. revalidatePath.

3. Create deals/buyers/page.tsx (server component):
   - Fetch buyers via getBuyers()
   - Show buyer count in header
   - Render BuyerList + BuyerIntakeForm components
   - `export const dynamic = "force-dynamic"`

4. Create buyer-list.tsx (client component):
   - Card layout for each buyer: name, phone (tel: link), email (mailto: link), buy box description
   - Show price range ($XXk - $XXk), funding type badge (Cash/Hard Money/Both), rehab tolerance badge
   - Target areas displayed as comma-separated text
   - "Edit" button toggles inline editing
   - "Deactivate" button with confirmation
   - Optional prop: `matchDealPrice?: number` — if provided, show green "Match" badge on buyers whose price range includes the deal price

5. Create buyer-intake-form.tsx (client component):
   - Form fields matching buyer schema: name (required), phone, email, buyBox (textarea), minPrice, maxPrice, fundingType (select: cash/hard_money/both), targetAreas (text), rehabTolerance (select: light/medium/heavy/any), notes (textarea)
   - Dollar inputs for min/max price with $ prefix
   - Submit calls createBuyer server action
   - Reset form on success
   - Form can also edit existing buyer (pass buyer prop to pre-fill)
