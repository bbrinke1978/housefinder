<task type="auto">
  <name>Task 2: MAO calculator + contract tracker tabs</name>
  <files>app/src/components/deal-mao-calculator.tsx, app/src/components/deal-contract-tracker.tsx</files>
  <action>
1. Create deal-mao-calculator.tsx ("use client"):
   - Input fields (controlled via useState, pre-filled from deal data):
     * ARV (integer, dollar input)
     * Rehab Estimate (integer, dollar input)
     * Wholesale Fee (integer, dollar input, default $15,000)
     * Offer Price (integer, dollar input — what you're offering the seller)
   - Auto-calculated fields (computed reactively, no DB roundtrip):
     * MAO = Math.round(ARV * 0.70 - Repairs - WholesaleFee)
     * Assignment Fee = the wholesaler's profit (user enters this directly or it defaults to $15k)
     * End Buyer All-In = Offer Price + Assignment Fee + Repairs
     * End Buyer Profit = ARV - End Buyer All-In
     * End Buyer ROI = Math.round((End Buyer Profit / End Buyer All-In) * 100)
     * Deal Score: "Great" if MAO > offerPrice + 10%, "Good" if MAO > offerPrice, "Tight" if MAO is within 5% of offerPrice, "Bad" if MAO < offerPrice
   - Sensitivity Analysis section (displayed as a simple table):
     * Row 1: "ARV 10% Lower" — recalculate MAO with arv * 0.90
     * Row 2: "Repairs 20% Higher" — recalculate MAO with repairs * 1.20
     * Row 3: "Both Worse" — arv * 0.90 AND repairs * 1.20
     * Each row shows: adjusted MAO, difference from base MAO, pass/fail indicator
   - "Save" button: calls updateDeal with arv, repairEstimate, wholesaleFee, mao, offerPrice
   - Format all dollar values with toLocaleString() and $ prefix
   - Color coding: green for positive profit/good deals, red for negative/bad deals, yellow for tight

2. Create deal-contract-tracker.tsx ("use client"):
   - Contract status progression displayed as a horizontal stepper/timeline:
     * Steps: Sent → Signed → In Escrow → Title Clear → Closing Scheduled
     * Current step highlighted, completed steps checked
     * Click a step to advance contract status (calls updateDeal with new contractStatus)
   - Earnest Money section:
     * Display amount (default $100)
     * "Refundable" badge (green if earnestMoneyRefundable is true)
     * Edit earnest money amount inline
   - Inspection Period section:
     * Display inspection deadline date
     * If deadline set: show countdown "X days remaining" using date-fns differenceInDays
     * If deadline not set: show date picker to set it (default 14 days from today using date-fns addDays)
     * Color: green if 7+ days, yellow if 3-6 days, red if < 3 days or past
   - Closing Date section:
     * Display closing date
     * Countdown to closing
     * Date picker to set/change
   - All updates call updateDeal server action with the changed fields
   - Show "No contract yet" message when contractStatus is null (deal not yet under contract)

IMPORTANT: All calculations are pure client-side math — no server roundtrips for computation. Only save to DB on explicit "Save" or field change confirmation. Use date-fns for all date operations. Format currency consistently.
