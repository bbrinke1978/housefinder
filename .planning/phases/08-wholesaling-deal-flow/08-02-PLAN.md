Create deal-actions.ts with "use server" directive:

1. `createDeal(formData: FormData)` — Parse form data with zod schema. Required: address, city. Optional: sellerName, sellerPhone, condition, timeline, motivation, askingPrice, arv, repairEstimate, wholesaleFee, propertyId. Auto-compute MAO if arv and repairEstimate provided (arv * 0.70 - repairEstimate - wholesaleFee). Insert into deals table. Auto-create status_change note "Deal created". revalidatePath("/deals"). Return { id: string }. Redirect to /deals/[id].

2. `updateDealStatus(dealId: string, status: string, note?: string)` — Validate with zod enum of DEAL_STATUSES. Get existing status first. Update deals.status and updatedAt. Insert status_change note with previousStatus/newStatus. If optional note provided, insert user note too. revalidatePath("/deals") and revalidatePath(`/deals/${dealId}`).

3. `updateDeal(dealId: string, data: FormData)` — Parse all editable deal fields. Update deals row. Auto-recompute MAO if arv/repairEstimate/wholesaleFee changed. revalidatePath. For seller qualification: condition, timeline, motivation, askingPrice are all optional text/integer fields updated directly.

All actions: auth() check first, zod/v4 validation, try/catch. Import `z` from "zod/v4".
