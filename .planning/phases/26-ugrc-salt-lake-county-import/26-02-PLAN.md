<objective>
Run the extended import script against the live production database — first as a dry-run, then for real — and capture the match/update counts plus a spot-check of the parcel-ID prefix-mismatch risk identified in the research (allowlist `07-`/`08-` vs Phase 25.5 `26-` prefix). This plan covers Phase 26 Success Criterion #1 ("UGRC data fetched and JOINed against existing Rose Park rows by parcel_id — log shows match count and updated count").

Purpose: Per project memory `feedback_no_dev_env.md`, we have no local dev environment. Verification = "deploy-then-observe or dry-run-on-prod" pattern. The dry-run flag built in Plan 01 makes prod-shadow safe (no writes). After dry-run confirms behavior, we run the real enrichment.

Output: Run output captured in 26-02-SUMMARY.md with match count, updated count, no-match count, and the prefix-distribution SQL result. The actual DB rows (if any) will have their UGRC fields populated via COALESCE — pre-existing non-null values are preserved.

Note on expected outcome: Per research Risk 2, Phase 25.5 dry-run showed zero accepted SLC NOD notices in the current Utah Legals window. That means there may be zero Rose Park rows in the DB at the time of this run. Phase 26 SUCCESS in that case is the script running cleanly, fetching the 8,270-parcel UGRC dataset, and logging "No match in our DB: 8270" — the script is a standing enrichment runner that becomes effective once Utah Legals produces real Rose Park rows.
</objective>
