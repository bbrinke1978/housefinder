1. **floor-plan-upload.tsx** ('use client'):
- Props: `{ dealId: string; propertyId?: string; onUploaded: () => void }`
- File input accepting PDF, JPG, PNG (accept=".pdf,.jpg,.jpeg,.png")
- Floor label selector: dropdown with Main, Upper, Basement, Garage, Other
- Version selector: radio/toggle for As-Is vs Proposed
- Client-side image compression for JPG/PNG: same resizeImage pattern as Phase 14 photo upload (1920px max, JPEG 0.8). PDFs accepted as-is up to 10MB.
- On submit: build FormData with file, dealId, floorLabel, version, sourceType='upload'. For images, also send naturalWidth/naturalHeight from Image() onload. For PDFs, send null dimensions. Call createFloorPlan server action.
- Show upload progress (simple loading spinner, not per-byte progress — FormData upload is atomic).
- Warn if file > 10MB for PDFs; block if > 10MB.

2. **floor-plan-viewer.tsx** ('use client'):
- Props: `{ plan: FloorPlanWithPins; readOnly?: boolean; dealId: string }`
- Import `react-zoom-pan-pinch` (TransformWrapper, TransformComponent). If peer dep conflict with React 19, fall back to simple CSS transform with touch-action: pinch-zoom.
- For image plans: render `<img src={plan.sasUrl}>` inside TransformComponent
- For PDF plans: render react-pdf `<Document><Page>` inside TransformComponent. Configure pdfjs worker as per research (pdfjs.GlobalWorkerOptions.workerSrc = new URL('pdfjs-dist/build/pdf.worker.min.mjs', import.meta.url).toString()). Render page 1 only with page navigation for multi-page PDFs.
- Pin overlay: position pins as absolutely-positioned DOM elements on top of the content div (inside TransformComponent). Each pin uses `left: ${pin.xPct * 100}%`, `top: ${pin.yPct * 100}%`, `transform: translate(-50%, -100%)`.
- Click-to-drop-pin: when not readOnly, clicking the content div opens the pin form at that position. Calculate xPct/yPct using getBoundingClientRect() on the inner content element. IMPORTANT: attach click handler to the inner content div inside TransformComponent, NOT the TransformWrapper — this ensures correct coordinates when zoomed/panned.
- Zoom controls: +/- buttons and reset button using TransformWrapper's zoomIn/zoomOut/resetTransform API
- Must be wrapped in `dynamic(() => import('./floor-plan-viewer'), { ssr: false })` at the tab level to avoid SSR issues with pdfjs and react-zoom-pan-pinch.

3. **floor-plan-pin.tsx** ('use client'):
- Props: `{ pin: FloorPlanPinRow; color: string; onDelete?: (id: string) => void; dealId?: string }`
- Renders a colored circle/marker icon using the PIN_COLORS palette
- Hover/tap shows a popover with: category label, note text, "View in Budget" link (if budgetCategoryId exists — navigates to `/deals/${dealId}?tab=financials&category=${budgetCategoryId}`)
- Delete button in popover (if onDelete provided, i.e., not readOnly)

4. **floor-plan-pin-form.tsx** ('use client'):
- Dialog/modal for creating a new pin
- Fields: category dropdown (all PinCategory values with color dots), note textarea, optional budgetCategoryId selector (if budget exists for this deal — passed as prop)
- On submit: call createPin server action with floorPlanId, xPct, yPct, category, note, budgetCategoryId
- Close dialog on success
