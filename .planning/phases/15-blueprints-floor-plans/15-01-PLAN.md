1. Add to schema.ts after propertyPhotos table:

**floorPlans table:**
- id: uuid PK defaultRandom
- dealId: uuid nullable FK to deals.id
- propertyId: uuid nullable FK to properties.id
- floorLabel: text notNull default 'main' (values: main|upper|basement|garage|other)
- version: text notNull default 'as-is' (values: as-is|proposed)
- sourceType: text notNull (values: upload|sketch)
- blobName: text nullable (for uploads)
- blobUrl: text nullable (for uploads)
- mimeType: text nullable (application/pdf|image/jpeg|image/png)
- sketchData: text nullable (JSON string for sketch rooms)
- naturalWidth: integer nullable (px, for coordinate normalization)
- naturalHeight: integer nullable (px, for coordinate normalization)
- totalSqft: integer nullable
- shareToken: text unique nullable
- shareExpiresAt: timestamp with timezone nullable
- sortOrder: integer notNull default 0
- createdAt: timestamp with timezone notNull defaultNow
- updatedAt: timestamp with timezone notNull defaultNow
- Indexes: idx_floor_plans_deal_id on dealId, idx_floor_plans_property_id on propertyId, idx_floor_plans_share_token on shareToken

**floorPlanPins table:**
- id: uuid PK defaultRandom
- floorPlanId: uuid notNull FK to floorPlans.id with onDelete cascade
- xPct: doublePrecision notNull (0.0 to 1.0)
- yPct: doublePrecision notNull (0.0 to 1.0)
- category: text notNull (plumbing|electrical|structural|cosmetic|hvac|roofing|flooring|painting|windows_doors|kitchen|bathroom|landscaping|general)
- note: text nullable
- budgetCategoryId: uuid nullable (no FK constraint — soft link to budget_categories)
- sortOrder: integer notNull default 0
- createdAt: timestamp with timezone notNull defaultNow
- Index: idx_floor_plan_pins_plan_id on floorPlanId

Export types: `FloorPlanRow = InferSelectModel<typeof floorPlans>`, `FloorPlanPinRow = InferSelectModel<typeof floorPlanPins>`

1. Create floor-plan-queries.ts with:
- `getFloorPlansByDeal(dealId: string): Promise<FloorPlanWithPins[]>` — fetch all floor plans for a deal, join pins, generate SAS URLs for upload-type plans. Order by sortOrder then floorLabel.
- `getFloorPlansByProperty(propertyId: string): Promise<FloorPlanWithPins[]>` — same but by propertyId
- `getFloorPlanWithPins(planId: string): Promise<FloorPlanWithPins | null>` — single plan with pins and SAS URL
- `getFloorPlanByShareToken(token: string): Promise<FloorPlanWithPins | null>` — for contractor view. Must validate shareExpiresAt > now(). Return null if expired or not found.
- `getFloorPlanCount(dealId: string): Promise<number>` — count for tab badge

All queries use `db.select()...from(floorPlans)` pattern with left join on floorPlanPins. Group pins into arrays in TypeScript (not SQL) — same pattern as other queries in the project. Generate SAS URLs using `generateFloorPlanSasUrl(plan.blobName)` when sourceType is 'upload' and blobName exists.

2. Create floor-plan-actions.ts with 'use server' directive:
- `createFloorPlan(formData: FormData): Promise<{ id: string }>` — handles both upload and sketch sourceType. For uploads: read file from formData, generate blobName as `{dealId}/{planId}-{floorLabel}.{ext}`, call uploadFloorPlanBlob, store blobName/blobUrl/mimeType. For sketch: store sketchData JSON string. Set naturalWidth/naturalHeight from formData (for uploads, read from image dimensions sent by client; for sketch, use canvas dimensions).
- `updateFloorPlan(formData: FormData): Promise<void>` — update floorLabel, version, sketchData, totalSqft. When totalSqft changes and dealId exists, also update deals.sqft with the sum of all floor plans' totalSqft for that deal.
- `deleteFloorPlan(planId: string): Promise<void>` — delete the plan (cascade deletes pins). If it was an upload, do NOT delete the blob (keep for safety). Recalculate deal sqft after deletion.
- `createPin(formData: FormData): Promise<{ id: string }>` — insert pin with floorPlanId, xPct, yPct, category, note, budgetCategoryId from formData
- `deletePin(pinId: string): Promise<void>` — delete a single pin
- `updatePin(formData: FormData): Promise<void>` — update pin category, note, budgetCategoryId
- `generateShareLink(planId: string): Promise<{ token: string; expiresAt: string }>` — generate crypto.randomUUID() token with 7-day expiry, update the floorPlans row. Return token and expiresAt.
- `revokeShareLink(planId: string): Promise<void>` — set shareToken and shareExpiresAt to null

All actions use revalidatePath('/deals/[id]') after mutations. Use the same 'use server' + FormData signature pattern as photo-actions.ts and contract-actions.ts.
