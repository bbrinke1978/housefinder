1. Install dependencies: `cd app && npm install react-konva konva`

2. **floor-plan-sketch.tsx** ('use client') — this is the core sketch tool:

Props: `{ dealId: string; planId?: string; initialRooms?: SketchRoom[]; floorLabel: string; version: string; readOnly?: boolean; onSave?: () => void }`

State architecture (per research):
- `rooms: SketchRoom[]` — document state, persisted
- `selectedId: string | null` — interaction state, ephemeral
- `stageScale: number` and `stagePos: { x, y }` — viewport state, session

Canvas setup:
- react-konva Stage with dynamic width (fill parent container via ResizeObserver or parent ref.offsetWidth) and fixed height (600px desktop, 400px mobile)
- Stage is draggable (pan) — disable when drawing/resizing a room
- Scroll wheel zoom using the verified Konva pattern from research: handleWheel reads pointer position, calculates mousePointTo, applies SCALE_STEP=1.1, enforces MIN_SCALE=0.25 / MAX_SCALE=4
- Mobile pinch zoom: Stage onTouchMove with two-finger distance calculation (per Konva mobile docs)
- Grid background: draw subtle grid lines (20px spacing at scale 1) using a Konva Layer with Line shapes. Grid helps with room alignment.

Room rectangles:
- Each room renders as a Konva Group containing: Rect (room outline, fill rgba(139,92,246,0.1), stroke #7c3aed 2px), Text (label + dimensions + sqft)
- Rooms are draggable. On dragEnd: update room.x, room.y in state. Use snap-to-grid via `dragBoundFunc`: round to nearest 20px.
- Selected room shows a Transformer with resize handles. rotateEnabled=false. boundBoxFunc enforces minimum 40px width/height.
- CRITICAL (from research pitfall 4): onTransformEnd must read `node.width() * node.scaleX()` and then reset scaleX/scaleY to 1. Then convert pixel dimensions back to feet using the scale factor (1px = 0.5ft by default, configurable).
- Room label text shows: "{label}\n{lengthFt}' x {widthFt}'\n{sqft} sq ft" — positioned at room.x+8, room.y+8
- Click empty area to deselect. Click room to select. Double-click room opens edit dialog.
- Delete key removes selected room (with confirmation).

Dimension tracking:
- Define a scale factor: PIXELS_PER_FOOT = 10 (so a 120px wide room = 12 feet)
- When room resized via Transformer: calculate new lengthFt = Math.round(newWidth / PIXELS_PER_FOOT * 10) / 10 and widthFt similarly. Update sqft = lengthFt * widthFt.
- Total sqft shown in toolbar: sum of all rooms' sqft values

Save:
- Save button serializes rooms array to JSON, builds FormData with sketchData, floorLabel, version, sourceType='sketch', totalSqft, naturalWidth=stage.width(), naturalHeight=stage.height(). Calls createFloorPlan (new) or updateFloorPlan (existing planId).
- Auto-save on blur/unmount is NOT implemented (explicit save only — consistent with the project pattern of explicit actions).

3. **sketch-room-dialog.tsx** ('use client'):
- Dialog for editing room properties: label (text input, e.g. "Living Room"), lengthFt (number input), widthFt (number input)
- Shows calculated sqft = lengthFt * widthFt
- Preset room labels: dropdown with common names (Living Room, Bedroom, Kitchen, Bathroom, Dining Room, Office, Closet, Hallway, Laundry, Storage) plus custom text input
- On save: updates the room in parent state. Room pixel dimensions recalculated from new feet values: width = lengthFt * PIXELS_PER_FOOT, height = widthFt * PIXELS_PER_FOOT.

4. **sketch-toolbar.tsx** ('use client'):
- Props: `{ onAddRoom: () => void; onSave: () => void; onZoomIn: () => void; onZoomOut: () => void; onResetView: () => void; totalSqft: number; isSaving: boolean; readOnly?: boolean }`
- Buttons: Add Room (+ icon), Save (disk icon), Zoom In (+), Zoom Out (-), Reset View
- Total sqft display: "Total: 1,234 sq ft" — prominent, updates reactively
- readOnly mode hides Add Room and Save buttons (for contractor view later)
