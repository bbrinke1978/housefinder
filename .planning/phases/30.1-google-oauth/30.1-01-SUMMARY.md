## Key Design Decisions

### 1. Client component → next-auth/react signIn

The existing login page is `"use client"`. The plan suggested a server-action wrapper `<form action={async () => { "use server"; ... }}>`, which cannot be inlined inside a client component. Solution: use `next-auth/react` `signIn("google", { redirectTo: "/" })`, which is the standard client-side approach and works identically. The behavior is the same — a redirect to Google's OAuth consent screen.

### 2. .gitignore exception for .env.local.example

`app/.gitignore` had `.env*` which blocked `.env.local.example`. Added `!.env.local.example` negation so the template is tracked in git. The real `.env.local` remains gitignored.

### 3. Pending badge uses inline Tailwind classes

The plan mentioned using the existing Badge component with custom yellow classes. The Pending badge is simpler as an inline `<span>` with `bg-yellow-100 text-yellow-800` Tailwind classes — no need to add a new Badge variant or import.
