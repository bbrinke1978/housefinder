  <name>Task 1: Add deduplicateSignals() to score.ts and call it in scoreProperty()</name>
  <files>scraper/src/scoring/score.ts</files>
  <action>
Add a pure `deduplicateSignals(signals: SignalInput[]): SignalInput[]` function above `scoreProperty()` in score.ts. The function collapses same-property signals of the same type filed within 90 days of each other, keeping only the EARLIEST one per 90-day window per signal type.

Deduplication rules:
- Only applies to signal types "nod" and "lis_pendens" (probate, code_violation, tax_lien, vacant are NOT deduplicated — each filing is distinct)
- For signals with null recorded_date, treat as a distinct signal (cannot determine if within 90 days)
- For signals with a sentinel 1970-01-01 date, treat as distinct (same as null logic)
- Group signals by signal_type, sort by recorded_date ascending
- Walk the sorted list: include a signal if it is >90 days after the last included signal of that type; otherwise skip it (it is a duplicate within the window)
- Return the deduplicated list merged back with all non-deduplicated signal types unchanged

Call `deduplicateSignals(activeSignals)` at the start of `scoreProperty()`, replacing the `activeSignals` variable that feeds the scoring loop. The function receives the already-filtered active signals array.

Export `deduplicateSignals` so dry-run.ts can use it directly if needed.

Do NOT change the function signature of `scoreProperty()` — it remains `(signals: SignalInput[], config: ScoringConfig): ScoreResult`.
