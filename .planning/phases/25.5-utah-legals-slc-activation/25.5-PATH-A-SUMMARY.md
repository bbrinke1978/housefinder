# Phase 25.5 Path A Addendum — SLC County-Wide Capture

## Background

Phase 25.5 shipped a strict 84116-only filter for SLC NOD notices (Option C-2 — UGRC parcel-allowlist). Dry-run revealed that ~145 SLC notices/week had parcels in `14-`, `20-`, `21-`, `22-`, `28-` prefix ranges (Sandy / Midvale / Holladay / Sugar House / etc.) — all genuinely outside 84116, all discarded by the filter.

Brian flagged this as wasted data: "can we add those cities in our list so we can filter and track them down also and decide if we want to move them into our deal section?"

After scoping conversation, Brian chose **Path A (tactical capture)**: extend the parcel-allowlist concept to all of Salt Lake County so each NOD can be tagged with its real neighborhood and surfaced in the dashboard. **Path B (strategic SLC pilot — re-tune scoring for urban density, per-neighborhood weights)** is deferred and tracked in TODO.md as a v1.4 candidate.

### 2. Replaced 84116-Only Filter with County-Wide Lookup

**Modified:** `scraper/src/sources/utah-legals.ts`

- Added `getSlcNeighborhoodMap()` that lazy-loads the JSON map from one of three candidate paths, caches as `Map<string, {zip, city}>`. Falls back to the 84116 Rose Park allowlist if the new map isn't generated yet (defense in depth).
- Replaced the strict 84116 allowlist filter with a parcel → neighborhood lookup. SLC notices are accepted if their normalized parcel ID is found in the 365k-parcel map; the matched entry's city + zip are tagged on the notice for downstream `normalizeCity()` to retag at upsert time.
- Old `getRoseParkAllowlist()` and `rose-park-parcel-allowlist.json` retained as fallback + historical reference.

### 3. Extended `normalizeCity()` for All SLC Neighborhoods

**Modified:** `scraper/src/lib/upsert.ts`

The `SLC_ZIP_TO_NEIGHBORHOOD` constant grew from one entry (84116 → Rose Park) to the full SLC zip table. `normalizeCity(city, zip)` now retags any property with an SLC zip to its proper neighborhood name (Sandy, Midvale, Sugar House, Holladay, Murray, Cottonwood Heights, etc.) at upsert time.

### 4. Production `target_cities` Updated

Updated `scraper_config.target_cities` row to include all 17 new SLC neighborhoods. **Hotfix applied:** the agent's update script initially clobbered the prior 16-city rural list; restored via merge script `restore-target-cities-merge.mjs`. Final count: **37 cities** (20 rural + 17 SLC) — verified in DB.

## Known Gap — Easy Follow-Up

48,635 parcels (~13%) fell into "Salt Lake County (other)" because their zips weren't in the hardcoded `SLC_ZIP_TO_NEIGHBORHOOD` table. The two big missing zips:

- **84081** — West Jordan / Magna area (~18,671 parcels)
- **84009** — South Jordan / Daybreak (~17,452 parcels)

Easy fix: add to the `SLC_ZIP_TO_NEIGHBORHOOD` map in `upsert.ts` and the mirror in `build-slc-parcel-neighborhood-map.ts`, then re-run the build script. Defer until Brian sees if these neighborhoods produce useful leads.
