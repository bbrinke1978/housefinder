  1. **Add `setVacantFlag` server action** to `app/src/lib/actions.ts`:
     - Signature: `async function setVacantFlag(propertyId: string, isVacant: boolean): Promise<void>`
     - Auth check (same pattern as other actions).
     - If `isVacant === true`: Insert into `distressSignals` with `signalType: "vacant"`, `status: "active"`, `recordedDate: new Date().toISOString().split("T")[0]`, `rawData: "Manual flag - field observation"`. Use `.onConflictDoNothing()` (the uq_distress_signal_dedup index handles duplicates).
     - If `isVacant === false`: Update distressSignals `SET status = 'resolved', resolvedAt = new Date()` WHERE `propertyId = X AND signalType = 'vacant' AND status = 'active'`.
     - Import `and` from drizzle-orm (already imported as `eq`).
     - `revalidatePath(\`/properties/${propertyId}\`)` after mutation.

  2. **Add `addManualSignal` server action** to `app/src/lib/actions.ts`:
     - Signature: `async function addManualSignal(propertyId: string, signalType: "probate" | "code_violation", rawData?: string): Promise<void>`
     - Auth check.
     - Zod validate: propertyId is uuid, signalType is one of "probate" | "code_violation".
     - Insert into distressSignals with `status: "active"`, `recordedDate: new Date().toISOString().split("T")[0]`, `rawData: rawData ?? "Manual entry"`, `sourceUrl: null`. Use `.onConflictDoNothing()`.
     - `revalidatePath(\`/properties/${propertyId}\`)`.

  3. **Add `getActiveVacantFlag` server action**:
     - Signature: `async function getActiveVacantFlag(propertyId: string): Promise<boolean>`
