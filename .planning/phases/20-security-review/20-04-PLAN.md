<task type="auto">
  <name>Task 1: Migrate Azure Functions app settings to Key Vault references</name>
  <files>
    .planning/phases/20-security-review/SECRETS-INVENTORY.md
  </files>
  <action>
    The Azure Functions app `housefinder-scraper` in resource group `rg-housefinder` already uses a Key Vault reference for WEBSITE_LEAD_API_KEY. Extend this pattern to ALL sensitive app settings.

    1. List current app settings to identify which secrets need migration:
       ```
       az functionapp config appsettings list --name housefinder-scraper --resource-group rg-housefinder --query "[].name" -o tsv
       ```

    2. For each of the following secrets, create a Key Vault secret (if it doesn't already exist) and update the Functions app setting to use a Key Vault reference:

       Secrets to migrate (skip WEBSITE_LEAD_API_KEY -- already done):
       - DATABASE_URL
       - RESEND_API_KEY
       - TRACERFY_API_KEY
       - TWILIO_ACCOUNT_SID
       - TWILIO_AUTH_TOKEN
       - TWILIO_PHONE_NUMBER

       For each secret:
       a. Read the current plaintext value:
          ```
          az functionapp config appsettings list --name housefinder-scraper --resource-group rg-housefinder --query "[?name=='SECRET_NAME'].value" -o tsv
          ```
       b. Store it in Key Vault (use lowercase-hyphenated naming convention to match existing pattern):
          ```
          az keyvault secret set --vault-name housefinder-kv --name "secret-name" --value "THE_VALUE"
          ```
       c. Get the secret URI:
          ```
          az keyvault secret show --vault-name housefinder-kv --name "secret-name" --query "id" -o tsv
          ```
       d. Update the Functions app setting to reference Key Vault:
          ```
          az functionapp config appsettings set --name housefinder-scraper --resource-group rg-housefinder --settings "SECRET_NAME=@Microsoft.KeyVault(SecretUri=https://housefinder-kv.vault.azure.net/secrets/secret-name/VERSION)"
          ```

       Key Vault secret naming convention (match existing patterns):
       - DATABASE_URL -> database-url
       - RESEND_API_KEY -> resend-api-key
       - TRACERFY_API_KEY -> tracerfy-api-key
       - TWILIO_ACCOUNT_SID -> twilio-account-sid
       - TWILIO_AUTH_TOKEN -> twilio-auth-token
       - TWILIO_PHONE_NUMBER -> twilio-phone-number

    3. Non-sensitive settings that should remain as plain app settings (do NOT migrate):
       - ALERT_EMAIL (not a secret -- it's an email address)
       - ALERT_PHONE_NUMBER (not a secret -- it's a phone number)
       - APP_URL (not a secret -- it's a URL)

    4. Verify all Key Vault references resolve correctly:
       ```
       az functionapp config appsettings list --name housefinder-scraper --resource-group rg-housefinder --query "[?contains(value, 'KeyVault')].{name:name, status:value}" -o table
       ```

    5. Test that the Functions app still works by triggering one of the timer functions manually or checking the Azure portal for function health.

    6. Update SECRETS-INVENTORY.md Section 2 (housefinder scraper) to reflect that secrets are now stored in Key Vault with references from Functions app settings. Update the "Value Location" column from "Azure Functions app settings" to "Azure Key Vault (housefinder-kv) via Functions reference".
  </action>
  <verify>
    Run: `az functionapp config appsettings list --name housefinder-scraper --resource-group rg-housefinder --query "[?contains(value || '', 'KeyVault')].name" -o tsv | wc -l` -- should return 7 (6 newly migrated + 1 existing WEBSITE_LEAD_API_KEY)
  </verify>
  <done>
    - All 7 sensitive Functions app settings use @Microsoft.KeyVault references
    - Non-sensitive settings (ALERT_EMAIL, ALERT_PHONE_NUMBER, APP_URL) remain as plain values
    - Key Vault secrets created with lowercase-hyphenated naming convention
    - Functions app still operational after migration
    - SECRETS-INVENTORY.md updated to reflect Key Vault locations
  </done>
