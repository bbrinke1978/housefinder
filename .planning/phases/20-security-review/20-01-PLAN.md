  <action>
    1. DELETE the file `app/src/app/api/migrate/route.ts` entirely. This is a live endpoint that seeds user accounts with a hardcoded password, gated only by the same API key used for leads ingest. Remove the entire `app/src/app/api/migrate/` directory.

    2. In `app/src/middleware.ts`, remove `api/migrate|` from the matcher regex. The current pattern is:
       `"/((?!api/auth|api/migrate|login|forgot-password|reset-password|sign|floor-plans|_next/static|_next/image|favicon.ico).*)"`
       Change to:
       `"/((?!api/auth|login|forgot-password|reset-password|sign|floor-plans|_next/static|_next/image|favicon.ico).*)"`

    3. In `app/src/lib/password-reset-actions.ts`, in the `resetPassword()` function, replace the single password length check:
       ```
       if (!password || password.length < 8) return { error: "Password must be at least 8 characters" };
       ```
       With three sequential checks:
       ```
       if (!password || password.length < 8) return { error: "Password must be at least 8 characters" };
       if (!/[A-Z]/.test(password)) return { error: "Password must contain at least one uppercase letter" };
       if (!/[0-9]/.test(password)) return { error: "Password must contain at least one number" };
       ```

    4. In `app/src/app/reset-password/page.tsx`, add a hint below the password input field showing the policy: "Minimum 8 characters, at least one uppercase letter and one number." Use `text-sm text-muted-foreground` styling. This is informational — server-side validation is the enforcement.

    5. Run `cd app && npm install next@15.5.15` to upgrade Next.js and patch the high-severity DoS CVE (GHSA-q4gf-8mx6-v5v3) and moderate disk cache CVE (GHSA-3x4c-7xq6-9pq8).

    6. Run `cd app && npm audit` and verify no high/critical vulnerabilities remain. Document any remaining medium/low findings.
  </action>

<task type="auto">
  <name>Task 2: Add security response headers to housefinder next.config.ts</name>
  <files>
    app/next.config.ts
    app/netlify.toml
  </files>
  <action>
    1. In `app/next.config.ts`, add an `async headers()` function to the nextConfig object. This is the recommended approach for Next.js on Netlify because `netlify.toml [[headers]]` don't reliably apply to SSR pages served by the @netlify/plugin-nextjs Edge Runtime adapter.

    Define security headers array:
    ```typescript
    const securityHeaders = [
      { key: "X-DNS-Prefetch-Control", value: "on" },
      { key: "Strict-Transport-Security", value: "max-age=63072000; includeSubDomains; preload" },
      { key: "X-Frame-Options", value: "SAMEORIGIN" },
      { key: "X-Content-Type-Options", value: "nosniff" },
      { key: "Referrer-Policy", value: "strict-origin-when-cross-origin" },
      { key: "Permissions-Policy", value: "camera=(), microphone=(), geolocation=(self)" },
      {
        key: "Content-Security-Policy-Report-Only",
        value: "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; img-src 'self' data: blob: https://*.blob.core.windows.net; font-src 'self' https://fonts.gstatic.com; connect-src 'self' https://api.mapbox.com https://events.mapbox.com https://*.tiles.mapbox.com; worker-src blob:; frame-ancestors 'none';"
      },
    ];
    ```

    Add to nextConfig:
    ```typescript
    async headers() {
      return [{ source: "/(.*)", headers: securityHeaders }];
    },
    ```

    Keep all existing config (transpilePackages, serverExternalPackages, images).

    Note: CSP is deployed as `Content-Security-Policy-Report-Only` first per research recommendation. After verifying no breakage on Netlify, a follow-up can promote it to enforcing `Content-Security-Policy`. The `unsafe-inline` and `unsafe-eval` in script-src are required by Next.js's client-side hydration and styled-jsx. The `worker-src blob:` is required by Mapbox GL JS.

    2. As belt-and-suspenders for static assets, add `[[headers]]` to `app/netlify.toml` AFTER the existing `[[plugins]]` section:
    ```toml
    [[headers]]
      for = "/*"
      [headers.values]
        X-Frame-Options = "SAMEORIGIN"
        X-Content-Type-Options = "nosniff"
        Referrer-Policy = "strict-origin-when-cross-origin"
        Strict-Transport-Security = "max-age=63072000; includeSubDomains; preload"
        Permissions-Policy = "camera=(), microphone=(), geolocation=(self)"
    ```

    Do NOT add CSP to netlify.toml — only the non-CSP headers. CSP requires careful tuning and should only come from next.config.ts.

    3. Build the app to verify headers config compiles: `cd app && npm run build`
  </action>
  <verify>
    Run: `cd app && grep -c "Strict-Transport-Security" next.config.ts && grep -c "X-Frame-Options" netlify.toml && npm run build 2>&1 | tail -3`
  </verify>

    - netlify.toml includes [[headers]] with 5 security headers for static asset coverage
    - App builds successfully with no compilation errors
  </done>
