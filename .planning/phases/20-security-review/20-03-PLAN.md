    Perform the full OWASP Top 10 audit using the checklist from the RESEARCH.md. This is a read-only code review — no code changes in this task. The output is the SECURITY-FINDINGS.md report.

    **A01: Broken Access Control**
    - Verify `/api/migrate` is deleted (should be done in Plan 01)
    - Check `/api/export` route has `auth()` at top
    - Check `/api/buyers/export` has `auth()` at top
    - Check `/api/contracts/[id]/pdf` has auth
    - Check `/api/leads` route — verify API key validation, confirm key is not the same value as any user-facing credential
    - Review `/sign/[token]` — confirm it only shows contract party info, no broader deal/property data leakage
    - Review `/floor-plans/[token]` — confirm 7-day expiry is enforced on token validation
    - Audit ALL server actions in `app/src/lib/` — grep for exported async functions and verify each has `auth()` at the top of mutating actions. Use: `grep -rn "export async function" app/src/lib/*.ts` and check each one.

    **A02: Cryptographic Failures**
    - Confirm password reset tokens use `randomBytes(32).toString('hex')` (already verified in research)
    - Confirm no plaintext passwords stored in DB or code
    - Verify `rejectUnauthorized: true` on DB SSL in `app/src/db/client.ts`
    - Search both repos for connection strings or API keys in source: `grep -rn "postgres://" app/src/ scraper/src/` and `grep -rn "sk_live\|sk_test\|Bearer " app/src/`

    **A03: Injection**
    - Find ALL `sql.raw()` calls: `grep -rn "sql\.raw" app/src/`
    - For each one, trace the argument back to its source. Verify it is a TypeScript constant, not user input.
    - Find ALL `db.execute(sql` calls: `grep -rn "db\.execute" app/src/`
    - For each one, verify parameterized values are used for user input.
    - Check nobshomes admin blog API for injection in slug generation.
    - Add inline code comments marking each `sql.raw()` usage with `// SECURITY: alias is a compile-time constant, not user input` or similar.

    **A05: Security Misconfiguration**
    - Confirm `/api/migrate` removed (Plan 01)
    - Confirm security headers added (Plans 01 + 02)
    - Check Azure Blob Storage container access levels — review `blob-storage.ts` for `publicAccessLevel` settings. Confirm all containers use `private` or no public access.

    **A06: Vulnerable Components**
    - Run `cd app && npm audit` and `cd /c/Users/bbrinkerhoff.WMCCAT/OneDrive/documents/nobshomes && npm audit`
    - Document all findings by severity. Confirm high/critical are fixed (Plan 01/02). Accept medium/low with rationale.
    - Note: `drizzle-kit` esbuild moderate CVE is dev-only — document as accepted risk.

    **A07: Identification and Authentication Failures**
    - Confirm password policy enforced (Plan 01)
    - Verify password reset tokens have 1-hour expiry and single-use marking
    - Verify JWT session maxAge setting in auth.ts
    - Document that rate limiting is skipped (3-user closed system, per research recommendation)

    **A09: Security Logging and Monitoring**
    - Check if auth failures are logged (NextAuth default behavior)
    - Document that centralized security alerting does not exist (acceptable at this scale)

    **Git History Secret Scan**
    - Attempt to install gitleaks: try `go install github.com/gitleaks/gitleaks/v8@latest` or download binary from GitHub releases. If neither Go nor Docker is available, fall back to manual grep: `git log --all --diff-filter=A -- "*.env*" "*.key" "*.pem"` and `git log --all -p --all -S "postgres://" -- "*.ts" "*.js" "*.json"` to find committed secrets.
    - Run on both housefinder and nobshomes repos.
    - If any leaked secrets are found, document them as CRITICAL findings requiring immediate rotation.

    **Write SECURITY-FINDINGS.md** following the structure from RESEARCH.md:
    ```

    **Audit Date:** 2026-04-XX
    **Scope:** housefinder + nobshomes repos

    ## Critical Findings (Must Fix Immediately)
    ## High Findings (Fixed in This Phase)
    ## Medium Findings (Accepted Risk / Documented)
    ## Low Findings (Accepted Risk)
    ## Closed / Not Applicable
    ## Recommendations
    ## Netlify Firewall Limitation (Documented)
    ```

    For each finding include: ID, title, severity, description, file/location, current status (fixed/accepted/deferred), and fix reference if applicable.

    Document the Netlify static IP limitation: full PostgreSQL firewall lockdown to Netlify IPs requires Enterprise tier. Current compensating controls: SSL enforcement (rejectUnauthorized: true), strong 30-char password, no public ports. Recommend partial lockdown by adding Azure Functions outbound IPs.
