  <name>Task 4: NextAuth gates (domain + active + roles)</name>
  <files>app/src/auth.ts</files>
  <action>
Per 29-RESEARCH.md "Auth changes":

1. In the `authorize` callback, after parsing email/password, reject any email not ending `@no-bshomes.com` (returns null — same response shape as "user not found" to avoid leaking info).
2. After the user lookup, reject if `!user.isActive`.
3. Reject if `(user.roles ?? []).length === 0`.
4. Add `roles: user.roles` to the returned user object so JWT can pick it up.

Update the `jwt` and `session` callbacks to thread `roles` and `userId` through:
```typescript
async jwt({ token, user }) {
  if (user) {
    token.roles = (user as any).roles ?? [];
    token.userId = (user as any).id;
  }
  return token;
},
async session({ session, token }) {
  if (session.user) {
    (session.user as any).id = token.userId;
    (session.user as any).roles = token.roles ?? [];
  }
  return session;
},
```

If a `next-auth.d.ts` type-augmentation file exists, extend `Session.user` and `JWT` with `id: string; roles: string[]`. If not, leave the `as any` casts (Phase 30 can clean up).

**Critical:** Test login flow before merging. Use brian@no-bshomes.com from a logged-out state. Confirm session.user.roles populates as ['owner'] in a server component.

   ```typescript
   if (!userCan(session.user.roles, "deal.edit_terms")) {
     throw new Error("Forbidden: insufficient role");
   }
   ```
   Use the Action mapping below. If an action involves multiple write surfaces, gate on the broadest required action.

2. **Audit log** AFTER the DB write succeeds:
   ```typescript
   await logAudit({
     actorUserId: session.user.id,
     action: "deal.terms_updated",
     entityType: "deal",
     entityId: dealId,
     oldValue: { offerPrice: prev.offerPrice, mao: prev.mao },
     newValue: { offerPrice: parsed.offerPrice, mao: parsed.mao },
   });
   ```
   For actions where old/new are clear, capture only the diff'd fields. For creates, oldValue=null. For deletes, newValue=null.

**Action → permission mapping (use as gate):**
- `createDeal`, `updateDeal` → `deal.edit_terms` (or `deal.create` for createDeal)
- `updateDealStatus` → `deal.edit_terms` if to lead/qualified/analyzed/offered/dead; `deal.edit_disposition` for marketing/assigned; `deal.edit_closing_logistics` for under_contract/closing/closed
- `addLeadNote`, `updateLeadStatus` → `lead.edit_status`
- `saveOwnerPhone`, `addManualContact` → `lead.edit_status` (any role that can edit lead status can add contacts)
- `runSkipTrace`, `runBatchSkipTrace` → `tracerfy.run`
- `createBuyer`, `updateBuyer`, `deactivateBuyer` → `buyer.create_or_edit`
- `deleteBuyer` (if exists) → `buyer.delete`
