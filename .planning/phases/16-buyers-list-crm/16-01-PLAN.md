<task type="auto">
  <name>Task 2: Types, queries, and server actions for buyer CRM</name>
  <files>app/src/types/index.ts, app/src/lib/buyer-queries.ts, app/src/lib/buyer-actions.ts</files>
  <action>
1. In types/index.ts, add:
   - `BuyerWithTags` — extends Buyer with `tags: string[]`, `followUpDate: string | null`, `lastContactedAt: Date | null`
   - `BuyerTimelineEntry` — `{ id: string; type: "comm_event" | "deal_interaction"; eventType?: string; status?: string; notes?: string | null; dealId?: string | null; dealAddress?: string | null; occurredAt: Date }`
   - `BuyerDealInteraction` — `{ id: string; buyerId: string; dealId: string; status: string; dealAddress: string; dealCity: string; createdAt: Date; updatedAt: Date }`
   - `BuyerWithMatchInfo` — extends Buyer with `tags: string[]`, `matchesArea: boolean`, `isFullMatch: boolean`
   - `OverdueBuyer` — `{ id: string; name: string; followUpDate: string }`

2. Create buyer-queries.ts ("use server" not needed — these are read queries):
   - `getBuyersForList(filters)` — fetch all buyers with tags aggregated (LEFT JOIN buyer_tags, GROUP BY buyer), supports search (name ILIKE), tag filter, status filter, target area filter, funding type filter. Returns BuyerWithTags[].
   - `getBuyerById(id)` — single buyer with tags. Returns BuyerWithTags | null.
   - `getBuyerTimeline(buyerId)` — parallel fetch comm events + deal interactions (with deal address/city via JOIN), merge, sort desc by date. Returns BuyerTimelineEntry[]. Mirror getLeadTimeline() pattern.
   - `getBuyerDealInteractions(buyerId)` — fetch buyer_deal_interactions joined with deals for address/city. Returns BuyerDealInteraction[].
   - `getAllBuyerTags()` — SELECT DISTINCT tag FROM buyer_tags ORDER BY tag. Returns string[]. For filter dropdown.
   - `getOverdueBuyerFollowups()` — buyers WHERE follow_up_date <= today AND is_active = true, ordered by follow_up_date, limit 10. Returns OverdueBuyer[].
   - `getBuyersForExport(filters?)` — same as getBuyersForList but returns flat rows suitable for CSV (tags joined as semicolon-separated string).
   - `getMatchingBuyersForDeal(dealPrice, dealCity)` — enhanced version of existing getMatchingBuyers: price range AND case-insensitive city match on target_areas. Returns BuyerWithMatchInfo[]. Include tags via left join.

3. Create buyer-actions.ts ("use server"):
   - `logBuyerCommEvent(formData: FormData)` — insert into buyer_communication_events, also update buyers.lastContactedAt. Zod validate: buyerId (uuid), eventType (enum), notes (optional string), dealId (optional uuid).
   - `setBuyerFollowUp(formData: FormData)` — update buyers.followUpDate. Zod validate: buyerId (uuid), followUpDate (date string or null to clear).
   - `addBuyerTag(formData: FormData)` — insert into buyer_tags with onConflictDoNothing on (buyerId, tag). Zod validate: buyerId (uuid), tag (string trim, 1-50 chars).
   - `removeBuyerTag(formData: FormData)` — delete from buyer_tags WHERE buyerId AND tag.
   - `updateBuyerDealInteraction(formData: FormData)` — upsert buyer_deal_interactions (onConflictDoUpdate on (buyerId, dealId) setting status + updatedAt). Zod validate: buyerId, dealId, status (enum).
   - `importBuyers(rows: Array<{name: string; phone?: string; email?: string; buyBox?: string; minPrice?: number; maxPrice?: number; fundingType?: string; targetAreas?: string; rehabTolerance?: string; notes?: string}>)` — batch insert with per-row try/catch, returns `{ imported: number; errors: string[] }`. Called directly (not FormData) via useTransition.
   - `logDealBlast(buyerId: string, dealId: string)` — insert buyer_communication_events with eventType "deal_blast" + upsert buyer_deal_interactions with status "blasted" (onConflictDoNothing). Also update buyers.lastContactedAt.

All actions use revalidatePath("/buyers") after mutation.
