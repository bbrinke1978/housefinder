  <action>
1. Create `scraper/src/lib/scraper-utils.ts`:
   - `delay(ms: number)`: Promise-based sleep for rate limiting
   - `launchBrowser()`: launches Playwright chromium in headless mode with a descriptive User-Agent: 'Mozilla/5.0 (compatible; housefinder-bot/1.0; +https://github.com/bbrinke1978/housefinder)'
   - `parseHeaderMap(page)`: extracts all `<th>` elements from a wpDataTable and returns a Map<string, number> mapping header text to column index. This is the CRITICAL defense against column reordering (see Pitfall 2 in research).
   - `classifyOwnerType(name: string)`: returns 'llc' if name contains LLC/Inc/Corp, 'trust' if contains Trust, 'estate' if contains Estate, else 'individual'

2. Create `scraper/src/lib/validation.ts`:
   - Define Zod schemas for scraped records:
   - `propertyRecordSchema`: z.object({ parcelId: z.string().min(1), address: z.string().min(1), city: z.string().min(1), ownerName: z.string().optional(), taxStatus: z.string().optional(), mortgageInfo: z.string().optional() })
   - `delinquentRecordSchema`: z.object({ parcelId: z.string().min(1), ownerName: z.string().optional(), year: z.string().optional(), amountDue: z.string().optional() })
   - Export both schemas and their inferred types

3. Create `scraper/src/sources/carbon-assessor.ts`:
   - Export async function `scrapeAssessor(): Promise<PropertyRecord[]>`
   - Use Playwright to navigate to `https://www.carbon.utah.gov/service/property-search/`
   - Wait for wpDataTable to render: `page.waitForSelector('.wpDataTable tbody tr', { timeout: 30000 })`
   - Use `parseHeaderMap` to build dynamic column index mapping
   - Extract property records: parcelId, address (from property address columns), city, ownerName, taxStatus, mortgageInfo (from available columns)
   - Handle pagination: wpDataTables typically has pagination controls. Check for `.paginate_button.next` and iterate through all pages with 1-2 second delay between pages (user's locked rate limiting decision).
   - Validate each record with Zod safeParse, log and skip invalid records
   - Close browser on completion (including in error path -- use try/finally)
   - IMPORTANT: Carbon County property search may require entering a search term to populate results. If the table is empty initially, try searching with a wildcard or browse-all approach. Log what the page looks like if no results appear.

4. Create `scraper/src/sources/carbon-delinquent.ts`:
   - Export async function `scrapeDelinquent(): Promise<DelinquentRecord[]>`
   - Navigate to `https://www.carbon.utah.gov/service/delinquent-properties/`
   - Wait for wpDataTable render
   - Use `parseHeaderMap` for dynamic column mapping
   - Extract: parcelId (Parcel column), ownerName (Name column), year (Year column), amountDue (Due column), propertyAddress (PropertyAddress + PropertyCity columns)
   - Handle pagination with 1-2 second delays
   - Validate with Zod, log and skip invalid
   - Close browser in finally block

Both scrapers should log: total records found, valid records after Zod validation, time elapsed. Use context.log if InvocationContext is available, else console.log.
