  <action>
1. Create `scraper/src/db/schema.ts` following the research schema exactly:
   - `signalTypeEnum`: pgEnum with values 'nod', 'tax_lien', 'lis_pendens', 'probate', 'code_violation', 'vacant'
   - `signalStatusEnum`: pgEnum with values 'active', 'resolved'
   - `ownerTypeEnum`: pgEnum with values 'individual', 'llc', 'trust', 'estate', 'unknown'
   - `properties` table: id (uuid PK defaultRandom), parcelId (text NOT NULL UNIQUE -- canonical dedup key), address (text NOT NULL), city (text NOT NULL), state (text NOT NULL default 'UT'), zip (text nullable), county (text NOT NULL), ownerName (text nullable), ownerType (ownerTypeEnum default 'unknown'), propertyType (text nullable), firstSeenAt (timestamp with tz defaultNow -- DATA-07), createdAt, updatedAt. Indexes on city and county.
   - `distressSignals` table: id (uuid PK), propertyId (uuid FK -> properties.id NOT NULL), signalType (signalTypeEnum NOT NULL), status (signalStatusEnum NOT NULL default 'active'), recordedDate (date nullable), sourceUrl (text nullable), rawData (text nullable -- JSON string of original scraped data), createdAt, resolvedAt (nullable). Indexes on propertyId, signalType. UNIQUE constraint on (propertyId, signalType, recordedDate) for deduplication.
   - `leads` table: id (uuid PK), propertyId (uuid FK -> properties.id NOT NULL UNIQUE -- one lead per property), status (text NOT NULL default 'new'), newLeadStatus (text NOT NULL default 'new'), distressScore (integer NOT NULL default 0), isHot (boolean NOT NULL default false), alertSent (boolean NOT NULL default false), firstSeenAt (timestamp), lastViewedAt (timestamp nullable), createdAt, updatedAt. Indexes on (isHot, status), newLeadStatus.
   - `scraperHealth` table: id (serial PK), county (text NOT NULL UNIQUE), lastRunAt, lastSuccessAt, lastResultCount (integer default 0), consecutiveZeroResults (integer NOT NULL default 0), updatedAt.
   - `scraperConfig` table: id (serial PK), key (text NOT NULL UNIQUE), value (text NOT NULL -- JSON-serialized), description (text nullable), updatedAt.

2. Create `scraper/src/db/client.ts`:
   - Import Pool from 'pg' and drizzle from 'drizzle-orm/node-postgres'
   - Import all schema tables with `import * as schema from './schema.js'`
   - Create Pool with: connectionString from process.env.DATABASE_URL, ssl: { rejectUnauthorized: true }, max: 3, idleTimeoutMillis: 300000, connectionTimeoutMillis: 5000
   - Export `db` as `drizzle({ client: pool, schema })`
   - Export `pool` for graceful shutdown

3. Create `scraper/drizzle.config.ts`:
   - schema: "./src/db/schema.ts"
   - out: "./src/db/migrations"
   - dialect: "postgresql"
   - dbCredentials: { url: process.env.DATABASE_URL! }

4. Create `.github/workflows/deploy-scraper.yml`:
   - Trigger: push to main, paths: 'scraper/**'
   - env: AZURE_FUNCTIONAPP_NAME: 'func-housefinder-scraper', AZURE_FUNCTIONAPP_PACKAGE_PATH: './scraper', NODE_VERSION: '20.x'
   - Job: build-and-deploy on ubuntu-latest
   - Steps: checkout@v4, setup-node@v4 with cache npm, "npm ci && npm run build" in scraper dir, install Playwright chromium ("npx playwright install --with-deps chromium"), Azure/functions-action@v1 with publish-profile secret AZURE_FUNCTIONAPP_PUBLISH_PROFILE
   - IMPORTANT: Include `npx playwright install --with-deps chromium` step BEFORE deploy -- the Azure Functions package must include the Playwright browser binary, OR set PLAYWRIGHT_BROWSERS_PATH in Azure App Settings and install on the host. For simplicity, install in CI and deploy the package with browsers.

NOTE: The .js extension in imports (e.g., './schema.js') is required for Node16 module resolution with TypeScript ESM output.
  </action>
