  <action>
1. Create `scraper/src/lib/upsert.ts`:

   **upsertProperty(record: PropertyRecord): Promise<string>**
   - Insert into properties table with parcelId, address, city, county: 'carbon', state: 'UT', ownerName, ownerType (via classifyOwnerType from scraper-utils)
   - ON CONFLICT (parcelId) DO UPDATE: set address, ownerName, ownerType, updatedAt
   - firstSeenAt is only set on initial insert (defaultNow in schema handles this)
   - Return the property UUID id

   **upsertSignal(propertyId: string, signal: { type: string; recordedDate?: string; sourceUrl?: string; raw?: unknown }): Promise<void>**
   - Insert into distressSignals: propertyId, signalType, status: 'active', recordedDate, sourceUrl, rawData: JSON.stringify(raw)
   - ON CONFLICT DO NOTHING (deduplication on property_id + signal_type + recorded_date)

   **upsertFromAssessor(records: PropertyRecord[]): Promise<{ upserted: number }>**
   - Iterate records, call upsertProperty for each
   - Return count of upserted properties

   **upsertFromDelinquent(records: DelinquentRecord[]): Promise<{ upserted: number; signals: number }>**
   - For each record: upsertProperty (to ensure property exists with the parcel), then upsertSignal with signalType 'tax_lien'
   - Return count of properties and signals upserted

   **upsertFromRecorder(records: RecorderRecord[]): Promise<{ upserted: number; signals: number }>**
   - For each record: upsertProperty (minimal data), then upsertSignal with the record's signalType ('nod' or 'lis_pendens')
   - Return counts

2. Create `scraper/src/lib/health.ts`:

   **updateScrapeHealth(params: { county: string; source: string; resultCount: number; success: boolean }): Promise<void>**
   - Upsert to scraperHealth table on county UNIQUE conflict
   - On success: set lastRunAt = now, lastSuccessAt = now, lastResultCount = resultCount
   - If resultCount === 0: increment consecutiveZeroResults
   - If resultCount > 0: reset consecutiveZeroResults to 0
   - On failure: set lastRunAt = now, do NOT update lastSuccessAt

   **checkHealthAlert(county: string): Promise<{ alert: boolean; consecutiveZeros: number }>**
   - Query scraperHealth for the county
   - If consecutiveZeroResults >= 3, return alert: true
   - Log at ERROR level if alert is true: "ALERT: {county} scraper has produced zero results for {N} consecutive runs. Manual investigation required."
  </action>

<task type="auto">
  <name>Task 2: Create dailyScrape timer trigger function</name>
  <files>
    scraper/src/functions/dailyScrape.ts
  </files>
  <action>
Create `scraper/src/functions/dailyScrape.ts` -- the Azure Functions v4 timer trigger that orchestrates the entire daily pipeline.

1. Import from @azure/functions: app, InvocationContext, Timer
2. Import all scrapers: scrapeAssessor, scrapeDelinquent, scrapeRecorder
3. Import upsert functions: upsertFromAssessor, upsertFromDelinquent, upsertFromRecorder
4. Import scoring: scoreAllProperties
5. Import health: updateScrapeHealth, checkHealthAlert
6. Import seed: seedDefaultConfig

7. Implement the handler function `dailyScrape(myTimer: Timer, context: InvocationContext)`:

   ```
   Step 1: Ensure default config exists (idempotent)
   await seedDefaultConfig()

   Step 2: Run scrapers independently (partial failure tolerance)
   Each scraper runs in its own try/catch. If assessor fails, delinquent still runs.

   const results = { assessor: null, delinquent: null, recorder: null }

   try {
     const assessorRecords = await scrapeAssessor()
     results.assessor = await upsertFromAssessor(assessorRecords)
     await updateScrapeHealth({ county: 'carbon', source: 'assessor', resultCount: assessorRecords.length, success: true })
   } catch (err) {
     context.error('Assessor scraper failed', err)
     await updateScrapeHealth({ county: 'carbon', source: 'assessor', resultCount: 0, success: false })
   }

   // Same pattern for delinquent and recorder

   Step 3: Score all properties (runs even if some scrapers failed)
   const scoreResults = await scoreAllProperties()

   Step 4: Health check
   await checkHealthAlert('carbon')

   Step 5: Log summary
   context.log('Daily scrape complete', {
     assessor: results.assessor,
     delinquent: results.delinquent,
     recorder: results.recorder,
     scoring: scoreResults,
     isPastDue: myTimer.isPastDue
   })
   ```

8. Register the timer:
   ```typescript
   app.timer('dailyScrape', {
     schedule: '0 0 5 * * *',  // 5 AM -- WEBSITE_TIME_ZONE=America/Denver makes this Mountain Time
     runOnStartup: false,       // NEVER true in production (see anti-patterns in research)
     handler: dailyScrape,
   });
   ```

IMPORTANT: Do NOT set runOnStartup: true. Research explicitly warns this fires on every scale-out event.

IMPORTANT: The NCRONTAB schedule '0 0 5 * * *' works with WEBSITE_TIME_ZONE=America/Denver set in Azure App Settings (which must be configured during deployment). Without that setting, this would be 5 AM UTC.
