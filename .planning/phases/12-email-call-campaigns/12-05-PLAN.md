**campaign-dispatch.ts** (core logic):
`dispatchCampaignEmails(): Promise<{sent: number; stopped: number; errors: number}>`:

1. **Auto-stop check:** Query campaignEnrollments with status='active', join with leads, join leads with deals (via deals.propertyId = leads.propertyId, nullable). Where deal.status IN ('closed', 'dead'): UPDATE enrollment status='stopped', stoppedAt=NOW(), stopReason='deal_closed'. Log count.

2. **Due enrollment query:** SELECT campaignEnrollments WHERE status='active' AND nextSendAt <= NOW() AND nextSendAt IS NOT NULL. Join with emailSteps (matching sequenceId and stepNumber = currentStep + 1 — the NEXT step to send), leads, properties, ownerContacts (for email).

3. **For each due enrollment:**
   a. Check emailSendLog for existing entry with this (enrollmentId, stepId) — if exists, skip (idempotency).
   b. Check deal status again (race condition protection per research pitfall 5).
   c. Load mail settings from scraperConfig (or RESEND_API_KEY env var fallback).
   d. Resolve merge fields: {firstName} from ownerName first word or "Homeowner", {address}, {city}, {senderName} from mail.fromName, {phone} from mail.phone.
   e. Insert emailSendLog with status='sent' (pre-log for idempotency).
   f. Send via Resend with idempotencyKey `${enrollmentId}-step-${stepNumber}`.
   g. Update emailSendLog with resendEmailId.
   h. Advance enrollment: increment currentStep. Calculate nextSendAt from the NEXT step's delayDays. If no more steps exist, set status='completed', nextSendAt=null.
   i. Insert contactEvent with eventType='emailed_client' for timeline tracking.
   j. On Resend error: update emailSendLog status to 'failed'. If 429 (quota), set status to 'quota_exceeded' and break loop (stop sending for today).

4. Return summary: {sent, stopped, errors}.

Use the same DB client pattern as existing scraper functions (import from shared db module). Use the same Resend pattern as scraper/src/alerts/email.tsx.

**campaignDispatch.ts** (Azure Function timer trigger):
Register a new timer trigger that runs daily at 5:15 AM MT (15 minutes after the main dailyScrape at 5:00 AM, so new leads are scored before campaigns process). Use the same pattern as dailyScrape.ts:
```

```
(5:15 AM MT = 12:15 UTC during MDT). Call `dispatchCampaignEmails()` and log results.
