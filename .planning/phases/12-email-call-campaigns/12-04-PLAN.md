**enrollment-actions.ts** ("use server"):

`enrollLeadInSequence(leadId: string, sequenceId: string): Promise<{success: true} | {error: string}>`:
1. Validate lead exists and has a contact email in ownerContacts (query for email not null). If no email, return error "Lead has no contact email".
2. Load mail settings via getMailSettings(). If resendApiKey is empty, fall back to RESEND_API_KEY env var. If neither exists, return error "Mail settings not configured".
3. Load the sequence + step 0.
4. In a DB transaction:
   a. Stop any existing active enrollment for this lead: UPDATE campaignEnrollments SET status='stopped', stoppedAt=NOW(), stopReason='re_enrolled' WHERE leadId AND status='active'.
   b. Insert new campaignEnrollments row with currentStep=0, status='active'. Calculate nextSendAt = NOW() + step[1].delayDays (if step 1 exists, else null — single-step sequence).
5. Resolve merge fields in step 0 subject + body: {firstName} from ownerName (split first word) or "Homeowner" fallback, {address} from properties.address, {city} from properties.city, {senderName} from mailSettings.fromName, {phone} from mailSettings.phone.
6. Insert into emailSendLog with status='sent' BEFORE sending (idempotency — if retry, check for existing log entry for this enrollmentId + stepId).
7. Send via Resend: `resend.emails.send({ from: "${fromName} <${fromEmail}>", to: ownerEmail, replyTo, subject: resolvedSubject, react: OutreachTemplate({ bodyHtml: resolvedBody, signature }), headers: { "X-Idempotency-Key": "${enrollmentId}-step-0" } })`.
8. Update emailSendLog with resendEmailId from response. If send fails, update status to 'failed' and return error.
9. Also insert a contactEvent with eventType='emailed_client' for the timeline.

`unenrollLead(enrollmentId: string): Promise<{success: true} | {error: string}>`:
- UPDATE campaignEnrollments SET status='stopped', stoppedAt=NOW(), stopReason='unenrolled' WHERE id=enrollmentId AND status='active'.

`bulkEnrollLeads(leadIds: string[], sequenceId: string): Promise<{enrolled: number; skipped: number; errors: string[]}>`:
- For each leadId, call enrollLeadInSequence. Collect results. Return summary of enrolled/skipped/errors. Process sequentially (not parallel) to respect Resend rate limits (5 req/sec). Add 200ms delay between sends.
