**contact-event-actions.ts:**
Create a "use server" module with:
- `logContactEvent(formData: FormData): Promise<{success: true} | {error: string}>` — validates leadId, eventType (must be one of 6 valid types), optional notes. Inserts into contactEvents table. Uses zod for validation consistent with existing pattern. Returns union type (not throw) for graceful client-side feedback per Phase 06 decision.

**contact-event-queries.ts:**
- `getLeadTimeline(leadId: string): Promise<TimelineEntry[]>` — queries contactEvents for this lead UNION with leadNotes for this lead, UNION with emailSendLog entries (joined to get step subject as label). Returns sorted descending by occurredAt. Each entry has: id, type (ContactEventType | "note" | "email_sent"), label (human-readable, e.g., "Called client", "Note", "Email: Subject line"), notes, occurredAt.
- `getLeadTouchpointCounts(leadIds: string[]): Promise<Map<string, number>>` — counts contactEvents per leadId for the given IDs. Returns a Map for efficient lookup. Uses `inArray` + `groupBy` pattern consistent with existing dashboard stats queries.

**contact-event-form.tsx:**
"use client" component. Renders a select dropdown with the 6 contact event types (using CONTACT_EVENT_LABELS from types), a textarea for optional notes, and a submit button. Uses `useActionState<{success:true}|{error:string}|null, FormData>` consistent with Phase 06 call log form pattern. On success, show a brief success indicator and reset form. Include the leadId as a hidden input.

**activity-timeline.tsx:**
"use client" component receiving `entries: TimelineEntry[]` as prop. Renders a vertical timeline (thin left border line with dots). Each entry shows: icon by type (Phone for called_client, Voicemail for left_voicemail, Mail for emailed_client, MessageSquare for sent_text, Users for met_in_person, Inbox for received_email, StickyNote for note, Send for email_sent), the label text, optional notes truncated to 2 lines with expand, and relative time via `date-fns formatDistanceToNow`. Use zinc/violet color tokens consistent with Phase 11 UI revamp.

**touchpoint-badge.tsx:**
Simple component: `TouchpointBadge({ count }: { count: number })`. If count === 0, return null. Otherwise render a small pill badge showing the count (e.g., "3") with a subtle icon. Use `text-xs bg-muted text-muted-foreground rounded-full px-1.5 py-0.5` styling.

**call-script-modal.tsx:**
"use client" component using @base-ui/react/dialog (consistent with Phase 11 command menu pattern — project uses @base-ui/react, not Radix). Props: ownerName, address, city, phone, distressScore, signalTypes. Renders a dialog with a pre-filled talk track script. The script text is stored in scraperConfig as "campaign.callScript" — fetch it via a query, fall back to a sensible default script like:

"Hi {firstName}, my name is {senderName}. I'm reaching out about the property at {address} in {city}. I work with homeowners who may be looking to sell quickly and I was wondering if you've considered selling? [Pause for response] I can make a fair cash offer and close on your timeline. Would you be interested in hearing more?"

Merge fields are resolved client-side from the passed props. Include a "Copy to clipboard" button for the resolved script text.
