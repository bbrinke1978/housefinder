**campaign-actions.ts** ("use server"):
- `createSequence(formData: FormData): Promise<{success: true; id: string} | {error: string}>` — validates name (required, 3-100 chars), creates emailSequences row, then creates emailSteps rows from a JSON-encoded steps array in formData. Each step has: stepNumber, delayDays, subject, bodyHtml. Step 0 has delayDays=0 (immediate). Use zod validation. Wrap in transaction.
- `updateSequence(formData: FormData): Promise<{success: true} | {error: string}>` — updates sequence name + isActive, deletes existing steps and re-inserts (simpler than diffing). Only allow update if no active enrollments reference deleted steps that have already sent.
- `deleteSequence(sequenceId: string): Promise<{success: true} | {error: string}>` — soft-delete by setting isActive=false. Do not hard-delete (enrollments reference it).

**campaign-queries.ts:**
- `getSequences(): Promise<EmailSequenceSummary[]>` — returns all sequences with: stepCount (count of emailSteps), activeEnrollments (count of campaignEnrollments where status='active'), totalSent (count of emailSendLog rows for this sequence's enrollments). Use subqueries or joins.
- `getSequenceWithSteps(sequenceId: string): Promise<{sequence: EmailSequenceRow; steps: EmailStepRow[]}>` — returns sequence + ordered steps for the editor.
- `getActiveEnrollments(sequenceId?: string): Promise<EnrollmentWithDetails[]>` — returns active enrollments joined with leads and properties to show ownerName, address, city, currentStep, totalSteps, status, nextSendAt. Optional filter by sequenceId.

**mail-settings-actions.ts** ("use server"):
- `getMailSettings(): Promise<MailSettings>` — reads from scraperConfig table using MAIL_SETTINGS_KEYS. Returns defaults for missing keys: fromName="" , fromEmail="", replyTo="", resendApiKey="" (or fall back to RESEND_API_KEY env var), phone="", signature="".
- `saveMailSettings(formData: FormData): Promise<{success: true} | {error: string}>` — validates all fields, upserts each key into scraperConfig using onConflictDoUpdate pattern (consistent with existing alert settings pattern in actions.ts).
